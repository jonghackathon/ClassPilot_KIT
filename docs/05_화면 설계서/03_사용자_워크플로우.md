## 1. 수강생 워크플로우

### WF-S1: 과제 제출

**시나리오:** 수강생이 홈 화면에서 미제출 과제를 확인하고 작성·제출한다.

```
┌──────────┐     ┌──────────┐     ┌──────────────┐     ┌──────────┐
│  수강생 홈 │────→│ 내 과제   │────→│  과제 작성     │────→│ 제출 완료  │
│          │     │  목록     │     │              │     │          │
│ "미제출   │     │ [미제출]탭│     │ 텍스트 입력    │     │ 성공 토스트│
│  과제" 카드│     │  선택    │     │ 자동저장(5분)  │     │ → 목록으로 │
└──────────┘     └──────────┘     │ AI사용 선택   │     └──────────┘
                                  │ [제출하기]     │
                                  └───────┬───────┘
                                          │
                                  ┌───────▼───────┐
                                  │  확인 모달      │
                                  │ "제출 후 수정   │
                                  │  불가합니다"    │
                                  │ [취소] [제출]   │
                                  └───────────────┘
```

**상세 단계:**

| # | 단계 | 화면 | 사용자 액션 | 시스템 반응 |
|---|------|------|-----------|-----------|
| 1 | 과제 확인 | 홈 | "제출할 과제" 카드 확인 | 미제출 과제 목록 표시 |
| 2 | 과제 선택 | 홈 또는 과제 목록 | [작성하기 →] 클릭 | 과제 작성 페이지 이동 |
| 3 | 내용 작성 | 과제 작성 | 텍스트 에디터에 답변 입력 | 글자수 실시간 표시 |
| 4 | 자동 저장 | 과제 작성 | (자동) | 5분마다 SubmissionHistory 저장, "자동저장 완료" 표시 |
| 5 | AI 사용 선택 | 과제 작성 | "사용함" 선택 시 사용 내용 입력 | 추가 입력란 표시 |
| 6 | 제출 | 과제 작성 | [제출하기] 클릭 | 확인 모달 표시 |
| 7 | 확인 | 확인 모달 | [제출] 클릭 | API 호출 → 성공 토스트 → 과제 목록 이동 |

**대안 흐름:**
- 4a: 자동저장 실패 → "저장 실패" 빨간 텍스트 표시, 수동 [임시저장] 유도
- 6a: [취소] → 모달 닫힘, 작성 계속
- 7a: API 실패 → 에러 토스트 "제출에 실패했습니다. 다시 시도해주세요"

### WF-S2: 복습 확인

**시나리오:** 수업 다음 날, 수강생이 AI 생성 복습 요약을 확인하고 퀴즈를 푼다.

```
┌──────────┐     ┌──────────┐     ┌──────────────────────┐
│  푸시 알림 │────→│  수강생 홈 │────→│  복습 상세             │
│ "복습 자료 │     │ "복습 자료 │     │                      │
│  도착!"   │     │  도착!" 카드│     │ 1. 오늘 배운 핵심 (3줄) │
└──────────┘     └──────────┘     │ 2. 퀴즈 (3문제)        │
                                  │ 3. 다음 수업 예습       │
                                  └──────────────────────┘
```

**상세 단계:**

| # | 단계 | 화면 | 사용자 액션 | 시스템 반응 |
|---|------|------|-----------|-----------|
| 1 | 알림 수신 | 외부 | 푸시 알림 탭 | 앱/웹 오픈 |
| 2 | 복습 진입 | 홈 | [확인하기 →] 클릭 | 복습 상세 페이지 이동, 읽음 처리 (PATCH /read) |
| 3 | 요약 읽기 | 복습 상세 | 핵심 요약 3줄 읽기 | - |
| 4 | 퀴즈 풀기 | 복습 상세 | 선택지 고르기 → [정답 확인] | 정답/오답 피드백 + 해설 표시 |
| 5 | 다음 문제 | 복습 상세 | [다음 문제 →] 클릭 | 다음 퀴즈 표시 (3문제 반복) |
| 6 | 예습 확인 | 복습 상세 | 다음 수업 예습 섹션 읽기 | - |

### WF-S3: 질문하기

**시나리오:** 수강생이 수업 내용에 대해 질문하고 AI 또는 강사의 답변을 받는다.

```
┌──────────┐     ┌──────────────────┐     ┌──────────────────┐
│ 질문하기   │────→│  AI 즉시 응답      │────→│  피드백 선택       │
│           │     │                  │     │                  │
│ 반 선택    │     │ AI가 1차 답변 생성  │     │ [도움됨 ✅]       │
│ 질문 입력  │     │ (교육과정 기반)     │     │   → 완료          │
│ [보내기]   │     │                  │     │                  │
└──────────┘     └──────────────────┘     │ [도움 안 됨 ❌]    │
                                          │   → 강사에게 전달   │
                                          │   → 답변 대기      │
                                          │   → 알림으로 통보   │
                                          └──────────────────┘
```

**상세 단계:**

| # | 단계 | 사용자 액션 | 시스템 반응 |
|---|------|-----------|-----------|
| 1 | 질문 입력 | 텍스트 입력 + [질문 보내기] | POST /api/ai/bot/questions |
| 2 | AI 응답 | - (대기) | AI가 교육과정 기반 1차 응답 생성 (2~3초) |
| 3 | 응답 확인 | AI 답변 읽기 | 답변 카드 표시 |
| 4a | 만족 | [도움됨 ✅] 클릭 | 질문 상태: 해결됨. FAQ 후보 등록 |
| 4b | 불만족 | [도움 안 됨 ❌] 클릭 | 강사에게 자동 전달. 강사 알림 발송 |
| 5 | 강사 답변 | - (대기) | 강사 답변 시 푸시 알림 |
| 6 | 답변 확인 | 알림 탭 → 질문 내역 | 강사 답변 표시 |

### WF-S4: 출결 확인

**시나리오:** 수강생이 자신의 출결 현황을 확인한다.

```
┌──────────┐     ┌──────────────────────┐
│ 바텀탭     │────→│  내 출결              │
│ [출결] 탭  │     │                      │
│           │     │ 출석률 바 (80%)        │
└──────────┘     │ 월간 달력 (상태 아이콘)  │
                 │ 최근 출결 리스트         │
                 └──────────────────────┘
```

> 수강생은 조회만 가능. 출결 수정 불가.

## 2. 강사 워크플로우

### WF-T1: 수업 진행 (출결 → 수업 → 정리)

**시나리오:** 강사의 하루 수업 사이클. 가장 핵심적인 워크플로우.

```
          ┌─────── 수업 전 ──────┐ ┌────── 수업 중 ───────┐ ┌────── 수업 후 ──────┐
          │                      │ │                      │ │                     │
┌────────┐│ ┌──────────────────┐ │ │ ┌──────────────────┐ │ │ ┌─────────────────┐ │
│ 강사 홈 │├→│  출결 입력         │ │ │ │  AI 코파일럿       │ │ │ │  녹음 업로드      │ │
│        │││                  │ │ │ │                  │ │ │ │                 │ │
│ 오늘수업│││ 반 자동 선택      │ │ │ │ 질문 입력 (텍/음성)│ │ │ │ 파일 업로드      │ │
│ 카드   │││ 학생별 버튼 탭    │ │ │ │ 4카드 SSE 생성    │ │ │ │ AI 요약 생성     │ │
│        │││ [전체출석]        │ │ │ │ [복사] [사용함]    │ │ │ │ [편집] 가능      │ │
└────────┘│ └──────────────────┘ │ │ └──────────────────┘ │ │ └─────────────────┘ │
          └──────────────────────┘ └──────────────────────┘ └─────────────────────┘
```

**수업 전: 출결 입력**

| # | 단계 | 사용자 액션 | 시스템 반응 |
|---|------|-----------|-----------|
| 1 | 진입 | 강사 홈 → [출결 입력] 클릭 | 출결 관리 페이지, 오늘 날짜 + 해당 반 자동 선택 |
| 2 | 개별 입력 | 학생별 [출석]/[지각]/[조퇴]/[결석] 버튼 탭 | 즉시 API 호출 (낙관적 업데이트), 버튼 색상 변경 |
| 3 | 일괄 처리 | [전체출석] 클릭 | 확인 모달 → 전원 PRESENT 일괄 처리 |
| 4 | 과제 체크 | 과제 O/X 버튼 탭 | Submission 상태 즉시 업데이트 |

**수업 중: AI 코파일럿**

| # | 단계 | 사용자 액션 | 시스템 반응 |
|---|------|-----------|-----------|
| 1 | 진입 | 강사 홈 → [AI 코파일럿 시작] 클릭 | 세션 생성 (POST /session), 반 구성·주제 자동 로드 |
| 2 | 질문 입력 | 학생 질문 텍스트 입력 또는 🎙 음성 입력 | 텍스트 변환 (Web Speech API) |
| 3 | 전송 | [→ 전송] 클릭 또는 Enter | SSE 스트리밍 시작 |
| 4 | 카드 생성 | - (대기) | 4카드 실시간 생성 (초보자/코드/심화/판서), 2~3초 |
| 5 | 활용 | [복사] 또는 [✓ 사용함] 클릭 | 클립보드 복사 / 사용 기록 저장 |
| 6 | 반복 | 다음 질문 입력 | 이전 질문 히스토리에 추가 |
| 7 | 종료 | [세션 종료] 클릭 | 확인 모달 → 세션 통계 요약 |

**수업 후: 녹음 정리**

| # | 단계 | 사용자 액션 | 시스템 반응 |
|---|------|-----------|-----------|
| 1 | 진입 | 더보기 → 녹음 정리 | 녹음 업로드 페이지 |
| 2 | 업로드 | 파일 선택 (mp3/m4a/wav) + 수업 선택 | 업로드 프로그레스 표시 |
| 3 | 처리 대기 | - | Whisper 변환 → Claude 요약 (비동기, 수분 소요) |
| 4 | 결과 확인 | [보기] 클릭 | 요약 상세: 다룬 개념 / 학생 질문 / 다음 수업 포인트 |
| 5 | 편집 | [편집] 클릭 | 요약 텍스트 수정 가능 |

### WF-T2: 과제 관리

**시나리오:** 강사가 과제를 등록하고, 제출 현황을 확인하고, 과정 이력을 검토한다.

```
┌────────────┐     ┌────────────┐     ┌──────────────────┐     ┌──────────────┐
│ 과제 목록   │────→│ 과제 상세   │────→│ 이력 타임라인      │     │ 피드백 작성   │
│            │     │            │     │ (모달)            │     │ (모달)       │
│ [+과제등록] │     │ 제출현황    │     │                  │     │             │
│            │     │ 테이블     │     │ 시간별 글자수 그래프│     │ 텍스트 입력   │
│ 반별 필터   │     │ [이력N회]  │────→│ 시점별 내용 미리보기│     │ [저장]       │
│ 마감일 정렬 │     │ [피드백]   │────→│ AI 사용 여부 표시  │     │             │
└────────────┘     └────────────┘     └──────────────────┘     └──────────────┘
```

**과제 등록 단계:**

| # | 단계 | 사용자 액션 | 시스템 반응 |
|---|------|-----------|-----------|
| 1 | 모달 열기 | [+ 과제 등록] 클릭 | 과제 등록 모달 표시 |
| 2 | 정보 입력 | 반 선택, 제목, 내용, 마감일 입력 | 실시간 유효성 검사 |
| 3 | 등록 | [등록] 클릭 | POST /api/assignments → 성공 토스트 → 목록 새로고침 |

**이력 검토 단계:**

| # | 단계 | 사용자 액션 | 시스템 반응 |
|---|------|-----------|-----------|
| 1 | 상세 진입 | [상세보기] 클릭 | 과제 상세: 제출률 바 + 학생별 테이블 |
| 2 | 이력 열기 | [12회] 이력 링크 클릭 | 타임라인 모달: 시간별 글자수 그래프 + 시점별 미리보기 |
| 3 | 시점 탐색 | [← →] 슬라이더 조작 | 해당 시점의 작성 내용 미리보기 |
| 4 | AI 확인 | AI 사용 여부·내용 확인 | - |
| 5 | 피드백 | [피드백 작성] 클릭 → 입력 → [저장] | 피드백 저장, 학생에게 알림 |

### WF-T3: 질문봇 관리

**시나리오:** 강사가 AI가 처리 못한 학생 질문에 답변하고 FAQ를 관리한다.

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────┐
│ 질문봇 관리   │────→│ AI 미해결 질문     │────→│ FAQ 관리      │
│              │     │                  │     │              │
│ [미답변] 탭   │     │ AI 응답 확인      │     │ 자동 등록된   │
│ [전체] 탭    │     │ 학생 피드백 확인   │     │ FAQ 편집·삭제 │
│ [FAQ] 탭     │     │ 강사 추가 답변     │     │              │
│              │     │ □ FAQ에 등록      │     │              │
└──────────────┘     └──────────────────┘     └──────────────┘
```

**상세 단계:**

| # | 단계 | 사용자 액션 | 시스템 반응 |
|---|------|-----------|-----------|
| 1 | 미답변 확인 | [미답변] 탭 선택 | 학생 피드백 "도움 안 됨" 질문 목록 |
| 2 | AI 응답 검토 | AI 응답 내용 읽기 | - |
| 3 | 답변 작성 | 추가 답변 입력 → [답변 전송] | POST /api/ai/bot/questions/[id]/answer |
| 4 | FAQ 등록 | □ "FAQ에 등록" 체크 | FAQ 목록에 자동 추가 |
| 5 | FAQ 관리 | [FAQ] 탭 → [편집] / [삭제] | FAQ 수정·삭제 |

## 3. 운영자 워크플로우

### WF-A1: 일일 운영 점검

**시나리오:** 운영자가 매일 아침 대시보드에서 전체 현황을 점검하고 필요한 조치를 취한다.

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────────┐
│  대시보드      │     │  드릴다운          │     │  액션             │
│              │     │                  │     │                  │
│ KPI 4개 확인  │────→│ 이탈 위험 → 레이더  │────→│ [연락하기]         │
│ 이탈 위험 확인 │     │ 미납 → 수강료 관리  │     │ [이탈 처리]        │
│ 오늘 수업 확인 │     │ 미제출 → 과제 현황  │     │ [납부 처리]        │
│ 미제출 확인   │     │ 학생 → 학생 상세   │     │ [출결 입력]        │
└──────────────┘     └──────────────────┘     └──────────────────┘
```

**매일 점검 체크리스트:**

| # | 점검 항목 | 대시보드 위치 | 이상 시 액션 |
|---|---------|------------|------------|
| 1 | 이탈 위험 학생 | 🔴 이탈 위험 StatCard + 상세 리스트 | [전체보기] → 이탈 레이더 → [연락하기] |
| 2 | 미납 수강료 | 💰 미납 StatCard | [수강료 관리] → [납부처리] |
| 3 | 오늘 수업 현황 | 오늘 수업 리스트 | 출결 미입력 반 확인 |
| 4 | 미제출 과제 | 미제출 과제 리스트 | 마감 지난 과제 확인 |

### WF-A2: 학생 등록 및 관리

**시나리오:** 새 학생을 등록하고 반에 배정한다.

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────┐     ┌──────────────┐
│ 학생 관리     │────→│  학생 등록 모달     │────→│ 학생 상세     │────→│ 반 배정       │
│              │     │                  │     │              │     │              │
│ [+학생 등록]  │     │ 이름·이메일·비번   │     │ 기본정보      │     │ 수강반 선택   │
│              │     │ 학년·학교        │     │ 탭: 출결/과제/ │     │ [수강 등록]   │
│              │     │ 학부모 연락처     │     │     상담/결제  │     │              │
│              │     │ 수강 반 선택     │     │              │     │              │
└──────────────┘     └──────────────────┘     └──────────────┘     └──────────────┘
```

**상세 단계:**

| # | 단계 | 사용자 액션 | 시스템 반응 |
|---|------|-----------|-----------|
| 1 | 모달 열기 | [+ 학생 등록] 클릭 | 등록 모달 표시 |
| 2 | 기본 정보 | 이름, 이메일, 임시 비밀번호 입력 | 이메일 중복 검사 |
| 3 | 프로필 | 학년, 학교, 메모 입력 | - |
| 4 | 학부모 | 학부모 이름, 전화번호 입력 | 전화번호 형식 검사, 암호화 저장 |
| 5 | 반 선택 | 수강할 반 체크 (복수 가능) | 정원 초과 시 경고 |
| 6 | 등록 | [등록] 클릭 | POST /auth/register → 학생 상세 페이지 이동 |

**유효성 검사:**

```
이메일: 필수, 이메일 형식, 중복 불가
이름: 필수, 2~20자
비밀번호: 필수, 8자 이상
학년: 선택
학교: 선택
학부모 전화번호: 010-XXXX-XXXX 형식
수강 반: 최소 1개 선택
```

### WF-A3: 이탈 관리

**시나리오:** 이탈 위험 학생을 발견하고 학부모에게 연락하여 상담하고, 결과를 기록한다.

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────────┐
│ 이탈 레이더   │────→│  학생 위험 상세     │────→│  연락 모달         │
│              │     │                  │     │                  │
│ 위험/주의 필터│     │ 위험도 70         │     │ 학부모 연락처 표시  │
│ 반별 필터    │     │ 요인별 상세 바     │     │ 상담 유형 선택     │
│              │     │ 위험도 추이       │     │ 상담 내용 입력     │
│              │     │                  │     │ 이전 상담 기록 표시 │
└──────────────┘     │ [상세] [연락] [이탈]│     │ [기록 저장]        │
                     └──────────────────┘     └──────────────────┘
                              │
                     ┌────────▼─────────┐
                     │  이탈 처리 모달    │  (실제 이탈 시)
                     │                  │
                     │ ⚠ 이탈 처리 확인  │
                     │ 이탈 사유 선택     │
                     │ AI 분석 결과 표시  │
                     │ [취소] [이탈 처리] │
                     └──────────────────┘
```

**연락~기록 단계:**

| # | 단계 | 사용자 액션 | 시스템 반응 |
|---|------|-----------|-----------|
| 1 | 위험 학생 확인 | 이탈 레이더에서 위험 학생 카드 확인 | 요인별 점수·상세 표시 |
| 2 | 연락 | [연락하기] 클릭 | 연락 모달: 학부모 정보 + 이전 상담 기록 표시 |
| 3 | 상담 | 전화/문자/대면 중 선택, 상담 내용 입력 | - |
| 4 | 기록 | [기록 저장] 클릭 | POST /api/consultations → 상담 기록 저장 |
| 5 | (이탈 시) | [이탈 처리] 클릭 | 이탈 모달: 사유 선택 + AI 분석 확인 |
| 6 | (이탈 확정) | 사유 선택 → [이탈 처리] | POST /api/ai/churn/events → 수강 해제 |

### WF-A4: 반 생성 및 운영

**시나리오:** 새 반을 생성하고 강사를 배정하고 학생을 등록한다.

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────────┐
│ 반 관리       │────→│  반 생성 모달      │────→│  반 상세           │
│              │     │                  │     │                  │
│ [+ 반 생성]   │     │ 반 이름           │     │ 수강생 목록        │
│              │     │ 과목·레벨         │     │ [+수강 등록]       │
│ 과목/요일/   │     │ 담당 강사         │     │ 시간표             │
│ 강사별 필터   │     │ 정원             │     │ 진도 현황           │
│              │     │ 수업 요일·시간     │     │                  │
│              │     │  └→ 충돌 자동 감지  │     │                  │
└──────────────┘     └──────────────────┘     └──────────────────┘
```

**시간표 충돌 감지:**

```
반 생성 시 수업 시간 입력:
  화요일 15:00~17:00

기존 시간표 확인:
  ✅ 충돌 없음 → 정상 생성
  ⚠ 충돌 발생 → "화 15:00~17:00에 (화-초4) 수업이 있습니다" 경고
     같은 강사 충돌 → 생성 차단
     같은 교실 충돌 → 경고 (생성 가능)
```

## 4. 공통 워크플로우

### WF-C1: 로그인

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────────┐
│  로그인 페이지 │────→│  인증 처리          │────→│  역할별 홈          │
│              │     │                  │     │                  │
│ 이메일 입력   │     │ NextAuth 검증     │     │ ADMIN → 대시보드   │
│ 비밀번호 입력 │     │ 세션 생성         │     │ TEACHER → 강사 홈  │
│ [로그인]     │     │ 역할 확인         │     │ STUDENT → 수강생 홈│
└──────────────┘     └──────────────────┘     └──────────────────┘
                              │
                     ┌────────▼─────────┐
                     │  인증 실패         │
                     │ "이메일 또는       │
                     │  비밀번호가        │
                     │  올바르지 않습니다" │
                     └──────────────────┘
```

### WF-C2: 알림 확인

```
모든 역할 공통:

[🔔 3] 클릭 → 알림 드롭다운 표시
  │
  ├── 알림 항목 클릭 → 해당 화면으로 이동
  │     예: "이탈 위험: 김민준" → 학생 상세 페이지
  │
  └── [모두 읽음] 클릭 → 알림 배지 숫자 초기화
```

**역할별 알림 유형:**

| 역할 | 알림 유형 |
|------|---------|
| ADMIN | 이탈 위험 도달, 미납 발생, 민원 접수 |
| TEACHER | 미답변 질문, 과제 제출, 코파일럿 세션 요약 |
| STUDENT | 복습 자료 도착, 과제 마감 임박, 강사 답변 도착 |

        │
        └── 세션 삭제 → /login 리다이렉트
