# AI 디자인 워크플로우

## ClassPilot 적용 사례

- **Step 1** — 요구사항 → Claude 화면설계서 작성 → Stitch로 시각화 → Figma 반입
- Figma MCP로 Claude가 디자인 컨텍스트 직접 읽어 Next.js + Tailwind 코드 변환
- 운영자 10개·강사 7개·학생 5개 총 22개 화면 이 흐름으로 초기 UI 구현

---

> ClassPilot 초기 디자인이 어떤 흐름으로 만들어졌는지 기록한다.  
> 이 방식은 Step 1 단계에서 완료된 작업이며, 이후 유사 프로젝트의 참고 레퍼런스로 남긴다.

---

## 전체 흐름 요약

```
요구사항 정의
    ↓
화면 설계서 작성 (Claude)
    ↓
디자인 AI로 시각화 (Stitch / Figma AI)
    ↓
Figma 반입
    ↓
Claude Figma MCP로 디자인 확정
    ↓
Next.js 코드 구현
```

## 3단계 — 디자인 AI로 시각화

**담당:** Stitch (Design with AI) / Figma AI  
**입력:** 화면 설계서 + `14_Figma_디자인_프롬프트.md`의 프롬프트

화면 설계서에서 추출한 구조와 디자인 시스템 토큰을 프롬프트로 변환해 디자인 AI에 투입한다.

### 프롬프트 구성 방식

```
[역할 설명]
학원 관리 SaaS의 [운영자/강사/학생] 화면입니다.

[디자인 시스템]
- Primary: Indigo-600 (#4f46e5)
- Background: Slate-50
- Font: Pretendard Variable
- Radius: rounded-2xl (16px), rounded-[30px] (30px)
- Card: glass-panel (white/82, backdrop-blur-18)

[화면 구성]
{해당 화면의 와이어프레임 내용}

[요청]
위 구조와 디자인 시스템을 따르는 화면 UI를 생성해주세요.
```

## 4단계 — Figma 반입

- 디자인 AI 결과물을 Figma에 가져온다
- 컴포넌트 정리, 레이아웃 조정, 색상 토큰 적용

---

## 5단계 — Claude Figma MCP로 확정

**담당:** Claude Code + Figma MCP 서버

Figma URL을 Claude에 제공하면 MCP를 통해 디자인 컨텍스트를 직접 읽는다.

```
# Figma MCP 활용 흐름
1. Figma URL → Claude에 전달
2. Claude가 get_design_context 호출
3. 디자인 구조, 색상, 간격 분석
4. Next.js + Tailwind 코드로 변환
5. 프로젝트 컨벤션(glass-panel 등)에 맞게 조정
```

→ `guides/claude-code/02_MCP_서버_활용.md`의 Figma MCP 섹션 참고

## 6단계 — Next.js 코드 구현

- Figma에서 확정된 디자인을 기반으로 Codex가 화면 구현
- 30개 페이지 UI 구현 완료
- 이후 Step 2~7에서 실데이터 연동

---

## 이 방식의 특징

**장점:**
- 요구사항 → 텍스트 설계 → 시각화 → 코드로 이어지는 일관된 흐름
- 각 단계에서 AI 도구가 역할에 맞게 투입됨
- Claude는 구조·판단, 디자인 AI는 시각화, Codex는 구현을 담당

**주의점:**
- 디자인 AI 결과물은 프로젝트 컨벤션과 맞지 않는 경우가 많음
- Figma MCP 이후 Tailwind 클래스 수동 조정이 필요한 경우 있음
- 화면 설계서가 충분히 구체적일수록 디자인 AI 결과 품질이 올라감
