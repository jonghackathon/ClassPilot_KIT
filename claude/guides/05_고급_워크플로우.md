# 고급 워크플로우 가이드

## 1. 멀티 에이전트 워크플로우

Claude Code는 서브에이전트를 생성하여 **병렬로 작업**을 수행할 수 있습니다.

### 서브에이전트 유형

| 유형 | 용도 | 도구 |
|------|------|------|
| `general-purpose` | 범용 (검색, 코드 실행, 파일 수정) | 모든 도구 |
| `Explore` | 코드베이스 탐색 (검색 전용) | 읽기/검색만 |
| `Plan` | 설계/계획 수립 | 읽기/검색만 |
| `code-reviewer` | 코드 리뷰 | 읽기/검색만 |

### 활용 패턴

#### 병렬 리서치
```
"프로젝트의 인증 시스템과 결제 시스템을 동시에 분석해줘"
→ Claude가 2개의 Explore 에이전트를 병렬 실행
```

#### 독립적 작업 분리
```
"이 3개 컴포넌트를 각각 리팩토링해줘"
→ 각 컴포넌트를 별도 에이전트에서 처리 (worktree 격리)
```

#### 세컨드 오피니언
```
"이 마이그레이션이 안전한지 독립적으로 검토해줘"
→ 별도 에이전트가 기존 분석과 무관하게 독립 리뷰
```

### Worktree 격리

서브에이전트에 `isolation: "worktree"`를 사용하면 독립적인 Git worktree에서 작업합니다.
- 메인 작업에 영향 없음
- 변경사항이 있으면 별도 브랜치로 보존
- 변경 없으면 자동 정리

## 2. 메모리 시스템

Claude Code는 대화 간 정보를 유지하는 **파일 기반 메모리 시스템**을 가지고 있습니다.

### 메모리 유형

| 유형 | 용도 | 예시 |
|------|------|------|
| `user` | 사용자 정보 | "시니어 풀스택 개발자, React 전문" |
| `feedback` | 작업 방식 피드백 | "테스트에서 DB 목킹 금지" |
| `project` | 프로젝트 상태 | "4/10까지 인증 기능 완료 필요" |
| `reference` | 외부 시스템 참조 | "버그 트래커는 Linear INGEST 프로젝트" |

### 메모리 저장/조회

```
# 저장 요청
"이 프로젝트에서 Prisma 마이그레이션 후 반드시 seed를 실행해야 한다는 걸 기억해줘"

# 조회
"이전에 이 프로젝트에 대해 기억하는 것 있어?"

# 삭제
"DB 관련 메모리를 지워줘"
```

### 메모리 저장 위치
```
~/.claude/projects/{프로젝트해시}/memory/
├── MEMORY.md          ← 인덱스 파일
├── user_role.md       ← 개별 메모리 파일
├── feedback_testing.md
└── project_deadline.md
```

## 3. 실행 모드

### 대화형 모드 (기본)
```bash
claude
```
터미널에서 대화하며 작업합니다.

### 원샷 모드
```bash
claude "이 프로젝트의 테스트를 실행하고 결과를 알려줘"
```
단일 질문/작업을 수행하고 종료합니다.

### 파이프 모드
```bash
cat error.log | claude -p "이 에러 로그를 분석해줘"
git diff | claude -p "이 변경사항을 리뷰해줘"
echo "TODO 주석을 모두 찾아줘" | claude -p
```
stdin 입력을 받아 비대화형으로 처리합니다.

### Headless 모드 (CI/CD용)
```bash
claude -p --output-format json "빌드 에러를 수정해줘"
```
JSON 출력으로 자동화 파이프라인에 통합합니다.

## 4. CI/CD 통합

### GitHub Actions 예시

```yaml
name: Claude Code Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Claude Code Review
        run: |
          npx @anthropic-ai/claude-code -p \
            "이 PR의 변경사항을 리뷰하고 이슈를 보고해줘" \
            --output-format json > review.json
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

### 자동 커밋 메시지 생성

```bash
# Git hook으로 활용 (.git/hooks/prepare-commit-msg)
#!/bin/sh
claude -p "git diff --cached의 내용을 기반으로 커밋 메시지를 생성해줘" > "$1"
```

## 5. 권한 관리

### 권한 모드

| 모드 | 설명 |
|------|------|
| **Default** | 읽기는 자동, 쓰기/실행은 승인 필요 |
| **Permissive** | 대부분 자동 승인 |
| **Restricted** | 모든 도구 사용 시 승인 필요 |

### 도구별 권한 설정

```json
// .claude/settings.json
{
  "permissions": {
    "allow": [
      "Read",
      "Glob",
      "Grep",
      "Bash(npm run test)",
      "Bash(npm run lint)"
    ],
    "deny": [
      "Bash(rm *)",
      "Bash(git push --force)"
    ]
  }
}
```

### 권한 설정 팁

- **개발 시**: 자주 쓰는 명령어를 `allow`에 추가하여 승인 피로 감소
- **CI/CD**: `--allowedTools` 플래그로 필요한 도구만 허용
- **보안**: `deny`에 위험한 패턴 추가

## 6. 플랜 모드

복잡한 작업 전에 구현 계획을 먼저 수립합니다.

```
# 대화 중 플랜 모드 진입
"이 기능을 구현하기 전에 계획부터 세워줘"

# 또는 Shift+Tab으로 Plan 모드 토글
```

### 플랜 모드의 장점
- 코드 수정 없이 설계만 먼저 리뷰
- 구현 전 접근방식 합의
- 대규모 리팩토링 시 안전망

## 7. 컨텍스트 관리

### `/compact` 활용
```
/compact                    # 전체 대화 압축
/compact 인증 관련만        # 특정 주제에 집중하여 압축
```

### 효과적인 컨텍스트 관리
1. **긴 대화는 주기적으로 `/compact`**: 토큰 절약 + 집중도 유지
2. **새 작업은 `/clear` 후 시작**: 이전 맥락 간섭 방지
3. **관련 파일만 참조**: 불필요한 파일 읽기 최소화
4. **CLAUDE.md로 반복 지시 대체**: 매번 설명 대신 파일로 관리

## 8. Agent SDK (프로그래밍 활용)

Claude Code를 프로그래밍 방식으로 사용할 수 있습니다.

```typescript
import { Claude } from "@anthropic-ai/claude-code";

const claude = new Claude({
  cwd: "/path/to/project",
});

// 대화형 사용
const result = await claude.chat("테스트를 실행해줘");
console.log(result.text);

// 스트리밍
for await (const event of claude.stream("코드를 리팩토링해줘")) {
  if (event.type === "text") {
    process.stdout.write(event.text);
  }
}
```

### 활용 사례
- 커스텀 코드 리뷰 봇
- 자동화된 코드 생성 파이프라인
- 프로젝트별 맞춤 AI 도구 개발
