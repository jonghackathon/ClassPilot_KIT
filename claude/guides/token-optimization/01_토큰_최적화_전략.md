# 토큰 최적화 및 재현성 전략

ClassPilot 개발에서 AI 토큰 비용을 줄이고, 유지보수성과 재현성을 높이기 위해 적용한 전략.

---

## 1. 계층적 문서 구조 (컨텍스트 피라미드)

```
CLAUDE.md          ← 매 세션 자동 로드 (최소 규칙만)
  ↓ 참조
AGENTS.md          ← 프레임워크 주의사항 (Next.js 16 등)
  ↓ 필요 시
/claude/docs/      ← 기획서, DB 스키마, API 명세 (필요할 때만 Read)
  ↓ 필요 시
/claude/guides/    ← 워크플로우 가이드 (작업 유형별 참고)
```

**효과**: Claude Code가 매번 전체 문서를 컨텍스트에 넣지 않아도 CLAUDE.md만으로 프로젝트 규칙을 파악. 세부 정보는 필요 시에만 참조하여 토큰 소비를 최소화.

---

## 2. `/compact` 활용 — 장기 세션 컨텍스트 압축

Step 6 AI 기능 구현처럼 하나의 세션에서 여러 파일을 수정하는 장기 작업 시:

```
작업 진행 → 컨텍스트 50% 이상 소비 → /compact → 핵심 결정만 유지 → 작업 계속
```

**적용 시점**: 대화가 길어져 컨텍스트 윈도우의 절반 이상을 사용했을 때.
**보완**: 중요한 의사결정은 `/compact` 전에 `/claude/done/` 문서로 기록하여 압축 후에도 유실 방지.

---

## 3. Codex 에이전트에 최소 컨텍스트 전달

```
❌ 잘못된 방식: "프로젝트 전체를 읽고 teacher 기능을 구현해줘"
✅ 올바른 방식: "src/app/api/attendance/ 경로에 GET/POST 엔드포인트를 만들고,
                Prisma 스키마의 Attendance 모델을 참조해서 CRUD를 구현해줘"
```

- Codex에는 해당 Step의 **구현 명세만** 전달
- 브랜치 단위로 작업 범위를 제한 (`codex/step4-teacher-assignments`)
- 전체 아키텍처 결정은 Claude Code가 사전에 완료

---

## 4. 프롬프트 템플릿 중앙 관리 (`/src/lib/ai/prompts.ts`)

모든 프로덕트 AI 기능의 프롬프트를 하나의 파일에서 관리:

| 함수 | 용도 |
|---|---|
| `buildCopilotAnswerPrompt()` | 수업 중 실시간 질문 답변 |
| `buildReviewGenerationPrompt()` | 복습 요약 + 퀴즈 생성 |
| `buildMonthlyReportPrompt()` | 월간 학부모 리포트 |
| `buildComplaintDraftPrompt()` | 민원 응답 초안 |

**유지보수 이점**:
- 프롬프트 수정 시 API 라우트 코드를 건드리지 않음
- `renderSections()` 헬퍼로 빈 필드 자동 필터링 → 불필요한 토큰 절약
- 타입 안전성 (`PromptSection`, 각 Input 타입) 으로 런타임 오류 방지

---

## 5. Fallback 메커니즘 — 재시도 토큰 낭비 방지

모든 AI API 호출에 fallback 함수를 구현:

```
Claude API 호출 → 성공 → 응답 파싱 → 사용
                → 실패 → fallback 기본 응답 반환 (재시도 없음)
                → 파싱 오류 → fallback 기본 응답 반환
```

**효과**: API 오류나 JSON 파싱 실패 시 무한 재시도 없이 사전 정의된 기본값을 반환. 토큰 낭비와 사용자 대기 시간을 동시에 줄임.

---

## 6. SWR 캐싱으로 중복 호출 제거

클라이언트에서 SWR의 자동 캐싱/재검증을 활용:

- 동일 API 키에 대한 중복 fetch 자동 방지
- 포커스 복귀 시 stale-while-revalidate로 최신 데이터 유지
- AI 생성 콘텐츠(리뷰 요약, 리포트)는 캐시 수명을 길게 설정하여 불필요한 재생성 방지

---

## 7. 문서 기반 재현성 — done/ 아카이브 시스템

```
/claude/done/
├── 09_구현_로드맵/           ← 전체 구현 계획과 진행 로그
├── 10_academy_backup/        ← 레거시 이관 기록
├── 11_백엔드_기능_구현_계획/ ← API 설계 및 완료 보고
├── 12_Supabase_연동/         ← 인프라 설정 기록
└── docs_history/             ← 설계 문서 이력
```

**재현성 보장**: 모든 Step의 구현 과정, 의사결정 근거, 워크플로우가 기록되어 있어:
- 새로운 AI 에이전트가 동일 패턴으로 후속 작업 가능
- 팀원 온보딩 시 별도 설명 없이 문서만으로 맥락 파악
- 유사 프로젝트에서 동일 워크플로우 재사용 가능

---

## ClassPilot 적용 사례

| 전략 | 적용 위치 | 효과 |
|---|---|---|
| 컨텍스트 피라미드 | CLAUDE.md → AGENTS.md → docs/ | 세션당 초기 컨텍스트 80% 절감 |
| /compact | Step 6 AI 기능 장기 세션 | 세션 분할 없이 연속 작업 |
| Codex 최소 지시 | Step 2~7 브랜치별 태스크 | 에이전트당 입력 토큰 최소화 |
| 프롬프트 중앙 관리 | prompts.ts 4개 빌더 함수 | 프롬프트 수정 1파일, 빈 필드 자동 제거 |
| Fallback | 모든 AI API 라우트 | 실패 시 재시도 0회, 즉시 기본값 |
| SWR 캐싱 | 프론트엔드 전체 | AI 콘텐츠 불필요한 재생성 방지 |
| done/ 아카이브 | 전 Step 완료 문서 | 후속 작업 컨텍스트 즉시 확보 |
