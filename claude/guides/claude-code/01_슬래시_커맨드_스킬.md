# 슬래시 커맨드 & 스킬 가이드

## 1. 내장 슬래시 커맨드

Claude Code에서 `/`를 입력하면 사용 가능한 커맨드 목록이 표시됩니다.

### 대화 관리

| 커맨드 | 설명 |
|--------|------|
| `/help` | 도움말 및 사용 가능한 커맨드 목록 |
| `/clear` | 대화 히스토리 초기화 |
| `/compact` | 대화 컨텍스트를 요약하여 압축 (토큰 절약) |
| `/compact [주제]` | 특정 주제에 집중하여 압축 |

### 설정 & 모드

| 커맨드 | 설명 |
|--------|------|
| `/config` | Claude Code 설정 열기 |
| `/permissions` | 도구 권한 설정 관리 |
| `/model` | 사용할 모델 변경 (opus, sonnet, haiku) |
| `/fast` | Fast 모드 토글 (동일 모델, 더 빠른 출력) |
| `/cost` | 현재 세션의 토큰 사용량 및 비용 표시 |

### Git & 코드 작업

| 커맨드 | 설명 |
|--------|------|
| `/commit` | 변경사항 분석 후 커밋 메시지 자동 생성 및 커밋 |
| `/review-pr [번호]` | PR 코드 리뷰 수행 |
| `/init` | 현재 프로젝트에 CLAUDE.md 파일 생성 |

### 파일 & 컨텍스트

| 커맨드 | 설명 |
|--------|------|
| `/add-dir [경로]` | 작업 디렉토리 추가 |
| `/terminal-setup` | 터미널 통합 설정 (shell 히스토리 공유 등) |

## 2. 스킬 (Skills)

스킬은 슬래시 커맨드를 통해 실행되는 확장 가능한 프롬프트 시스템입니다.

### 내장 스킬

#### `/commit` — Git 커밋
```
/commit
```
- staged/unstaged 변경사항 자동 분석
- 최근 커밋 스타일에 맞는 메시지 생성
- Co-Authored-By 태그 자동 추가

#### `/review-pr` — PR 리뷰
```
/review-pr 123
/review-pr            # 현재 브랜치의 PR 리뷰
```
- 코드 품질, 보안, 성능 관점에서 리뷰
- 구체적인 개선 제안 포함

#### `/simplify` — 코드 단순화
```
/simplify
```
- 변경된 코드의 재사용성, 품질, 효율성 검토
- 발견된 이슈 자동 수정

#### `/loop` — 반복 실행
```
/loop 5m /commit       # 5분마다 커밋
/loop 10m "테스트 실행"  # 10분마다 특정 작업
/loop                   # 기본 10분 간격
```
- 주기적 작업 자동화에 유용
- 배포 모니터링, 반복 빌드 등

#### `/schedule` — 원격 에이전트 스케줄링
```
/schedule
```
- cron 기반 원격 에이전트 생성/관리
- 자동화된 주기적 작업 실행

#### `/init` — CLAUDE.md 생성
```
/init
```
- 프로젝트 구조 분석 후 CLAUDE.md 자동 생성
- 빌드/테스트 커맨드, 코드 스타일 가이드 포함

## 3. 커스텀 슬래시 커맨드 만들기

### 방법 1: Markdown 파일 기반

`.claude/commands/` 디렉토리에 `.md` 파일을 생성하면 커스텀 커맨드가 됩니다.

```
프로젝트 루트/
└── .claude/
    └── commands/
        ├── deploy.md        → /deploy
        ├── test-all.md      → /test-all
        └── db-migrate.md    → /db-migrate
```

#### 예시: `.claude/commands/deploy.md`

```markdown
배포 전 체크리스트를 수행하고 배포합니다.

1. `npm run build`를 실행하여 빌드 성공 확인
2. `npm test`를 실행하여 모든 테스트 통과 확인
3. git status로 커밋되지 않은 변경사항 확인
4. 모두 통과하면 `npm run deploy` 실행
5. 배포 결과를 요약하여 보고
```

#### 예시: `.claude/commands/review.md`

```markdown
현재 브랜치의 변경사항을 코드 리뷰합니다.

다음 관점에서 리뷰해주세요:
- 보안 취약점 (SQL injection, XSS 등)
- 성능 이슈
- 코드 중복
- 에러 핸들링 누락
- 타입 안전성

발견된 이슈를 심각도별로 정리하고, 구체적인 수정 제안을 포함해주세요.
```

### 방법 2: 프로젝트 레벨 vs 사용자 레벨

| 위치 | 범위 | 공유 |
|------|------|------|
| `프로젝트/.claude/commands/` | 프로젝트 전용 | Git으로 팀과 공유 가능 |
| `~/.claude/commands/` | 모든 프로젝트에서 사용 | 개인 전용 |

### 커맨드 파일 작성 팁

1. **명확한 지시**: Claude가 수행할 단계를 구체적으로 나열
2. **도구 활용 명시**: "터미널에서 실행", "파일을 읽어서" 등 행동 지시
3. **출력 형식 지정**: 결과를 어떤 형태로 보여줄지 명시
4. **변수 사용**: `$ARGUMENTS`로 사용자 입력값 받기

#### `$ARGUMENTS` 활용 예시

```markdown
<!-- .claude/commands/find-usage.md -->
$ARGUMENTS로 전달된 함수/클래스의 사용처를 프로젝트 전체에서 검색합니다.

1. grep으로 사용처 검색
2. 파일별로 그룹화하여 정리
3. 각 사용처의 컨텍스트(전후 3줄) 포함
```

사용법:
```
/find-usage getUserById
```

## 4. 실용적인 커스텀 커맨드 예시

### `/db-reset` — DB 초기화
```markdown
데이터베이스를 초기화하고 시드 데이터를 넣습니다.

1. `npx prisma migrate reset --force` 실행
2. `npx prisma db seed` 실행
3. 결과를 확인하고 보고
```

### `/check` — 전체 검증
```markdown
코드 품질을 전체적으로 검증합니다.

1. TypeScript 컴파일 확인: `npx tsc --noEmit`
2. ESLint 검사: `npm run lint`
3. 테스트 실행: `npm test`
4. 빌드 확인: `npm run build`

각 단계의 결과를 요약하고, 실패한 항목이 있으면 수정을 제안해주세요.
```

### `/summarize-pr` — PR 요약
```markdown
현재 브랜치의 변경사항을 PR 설명문 형태로 요약합니다.

1. `git diff main...HEAD`로 변경사항 확인
2. 변경된 파일과 커밋 히스토리 분석
3. 다음 형식으로 PR 설명 작성:

## Summary
- 주요 변경사항 3줄 이내

## Changes
- 변경된 파일별 설명

## Test Plan
- 테스트 방법
```
