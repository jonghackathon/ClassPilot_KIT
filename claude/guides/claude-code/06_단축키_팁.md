# 단축키 & 생산성 팁

## 1. 키보드 단축키

### CLI 단축키

| 단축키 | 기능 |
|--------|------|
| `Enter` | 메시지 전송 |
| `Shift+Tab` | Plan 모드 ↔ Code 모드 전환 |
| `Ctrl+C` | 현재 작업 중단 |
| `Ctrl+D` | 세션 종료 |
| `Esc` | 현재 입력 취소 / 작업 중단 |
| `↑` / `↓` | 이전/다음 메시지 탐색 |
| `Tab` | 자동완성 (파일명, 커맨드) |

### VS Code 확장 단축키

| 단축키 | 기능 |
|--------|------|
| `Cmd+Shift+P` → "Claude" | Claude Code 명령어 목록 |
| `Cmd+L` | Claude 패널 포커스 (기본) |
| 코드 선택 → `Cmd+L` | 선택한 코드를 Claude에 전송 |

### 커스텀 키바인딩

`~/.claude/keybindings.json`에서 단축키를 커스터마이즈할 수 있습니다:

```json
[
  {
    "key": "ctrl+s",
    "command": "submit"
  },
  {
    "key": "ctrl+shift+c",
    "command": "compact"
  }
]
```

## 2. 생산성 팁

### 프롬프트 작성 팁

#### 구체적으로 지시하기
```
❌ "이 코드를 개선해줘"
✅ "UserProfile 컴포넌트에서 API 호출을 SWR로 변경하고, 로딩 상태를 Skeleton UI로 교체해줘"
```

#### 컨텍스트 제공하기
```
❌ "에러가 나"
✅ "npm run build 시 'Cannot find module @/lib/auth' 에러 발생.
    src/app/api/auth/route.ts에서 import 경로 문제인 것 같아"
```

#### 단계적으로 요청하기
```
1단계: "먼저 현재 인증 시스템 구조를 분석해줘"
2단계: "OAuth 추가를 위한 계획을 세워줘"
3단계: "계획대로 구현해줘"
```

### 효율적인 작업 패턴

#### 파일 참조
대화에 파일을 직접 참조하여 컨텍스트를 명확하게:
```
src/lib/auth.ts의 getSession 함수를 수정해줘
```

#### 이미지 활용
스크린샷을 붙여넣어 UI 문제를 설명:
```
[스크린샷 붙여넣기] 이 버튼의 정렬이 깨져있어. 수정해줘.
```

#### Git diff 활용
```bash
git diff | claude -p "이 변경사항에 문제가 있는지 리뷰해줘"
```

## 3. 자주 쓰는 명령 패턴

### 빠른 디버깅
```
"npm run build 에러를 수정해줘"
→ Claude가 빌드 실행 → 에러 분석 → 수정 → 재빌드 확인
```

### 빠른 리팩토링
```
"/simplify"
→ 최근 변경된 코드를 자동 분석하고 개선
```

### 빠른 커밋
```
"/commit"
→ 변경사항 분석 → 커밋 메시지 생성 → 커밋
```

### PR 생성
```
"develop 브랜치로 PR 만들어줘"
→ 변경사항 요약 → PR 생성 → URL 반환
```

### 코드 검색
```
"getUserById 함수가 어디서 사용되는지 찾아줘"
→ 프로젝트 전체 검색 → 사용처 목록 + 컨텍스트
```

## 4. 상황별 베스트 프랙티스

### 새 기능 개발 시
```
1. /init으로 CLAUDE.md 확인/생성
2. Plan 모드로 설계 논의 (Shift+Tab)
3. 구현 진행
4. /simplify로 코드 품질 검토
5. /commit으로 커밋
```

### 버그 수정 시
```
1. 에러 메시지/스크린샷 제공
2. "이 에러를 분석하고 수정해줘"
3. Claude가 원인 분석 → 수정 → 테스트 실행
4. /commit
```

### 코드 리뷰 시
```
1. /review-pr [PR 번호]
2. 또는: git diff main...HEAD | claude -p "리뷰해줘"
```

### 대규모 리팩토링 시
```
1. Plan 모드에서 전략 수립
2. 작업을 단계별로 분리
3. 각 단계마다 테스트 실행 확인
4. /compact으로 컨텍스트 관리
```

## 5. 트러블슈팅

### "컨텍스트가 너무 길어요"
```
/compact        # 대화 압축
/clear          # 새로 시작
```

### "Claude가 잘못된 방향으로 가요"
```
Esc             # 현재 작업 중단
"아니, 그 방향이 아니라 [구체적 지시]"
```

### "도구 권한이 거부됐어요"
```
/permissions    # 권한 설정 확인 및 수정
```

### "MCP 서버가 연결 안 돼요"
```bash
claude mcp list           # 서버 상태 확인
claude mcp remove [이름]  # 제거 후 재추가
claude mcp add [이름] -- [명령어]
```

### "느려요"
- `/fast` 토글로 Fast 모드 활성화
- 불필요한 MCP 서버 비활성화
- `/compact`로 컨텍스트 크기 줄이기

## 6. 숨겨진 유용한 기능

1. **멀티 디렉토리**: `/add-dir`로 여러 프로젝트를 동시에 작업
2. **이미지 분석**: 스크린샷을 붙여넣으면 UI 문제 자동 분석
3. **PDF 읽기**: PDF 파일 경로를 주면 내용 분석 가능
4. **Jupyter 노트북**: `.ipynb` 파일 읽기/편집 지원
5. **Git 기록 분석**: "최근 커밋에서 버그가 유입된 것 같아" → git log/blame 활용
6. **자동 메모리**: 중요한 피드백을 자동으로 기억하여 다음 대화에 반영
7. **비용 추적**: `/cost`로 현재 세션 비용 실시간 확인
