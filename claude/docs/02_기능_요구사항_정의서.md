## 1. 요구사항 표기 규칙

| 표기 | 의미 |
|------|------|
| `MUST` | 반드시 구현. 없으면 서비스 불가 |
| `SHOULD` | 구현 권장. 없어도 동작하지만 품질 저하 |
| `NICE` | 여유 있을 때 추가. MVP 이후 고려 |
| ★ | MVP Phase 1 포함 대상 |

## 2. 사용자 역할 정의

| 역할 | 설명 | 주요 접근 범위 |
|------|------|---------------|
| `ADMIN` | 학원 운영자 (원장·데스크) | 전체 데이터 읽기·쓰기 |
| `TEACHER` | 강사 | 담당 반·학생 데이터 |
| `STUDENT` | 수강생 | 본인 데이터만 |

## 3. 인증 · 권한

### 3.1 인증

| ID | 요구사항 | 우선순위 | Phase |
|----|----------|----------|-------|
| AUTH-01 | 이메일 + 비밀번호 로그인 `MUST` | 높음 | ★ 1 |
| AUTH-02 | 로그인 시 역할(ADMIN / TEACHER / STUDENT) 분기 `MUST` | 높음 | ★ 1 |
| AUTH-03 | JWT 기반 세션 관리, 만료 시 자동 로그아웃 `MUST` | 높음 | ★ 1 |
| AUTH-04 | 비밀번호 재설정 (이메일 링크) `SHOULD` | 중간 | 2 |
| AUTH-05 | 소셜 로그인 (Google) `NICE` | 낮음 | 3 |

### 3.2 권한

| ID | 요구사항 | 우선순위 | Phase |
|----|----------|----------|-------|
| AUTH-06 | 역할별 접근 가능 페이지·API 분리 `MUST` | 높음 | ★ 1 |
| AUTH-07 | TEACHER는 담당 반 외 학생 데이터 접근 불가 `MUST` | 높음 | ★ 1 |
| AUTH-08 | STUDENT는 본인 데이터만 조회 가능 `MUST` | 높음 | ★ 1 |
| AUTH-09 | ADMIN은 강사·학생 계정 생성·비활성화 가능 `MUST` | 높음 | ★ 1 |

### 3.3 예외 처리

- 권한 없는 페이지 접근 시 → 역할별 홈으로 리다이렉트
- 세션 만료 상태에서 API 호출 시 → 401 반환 후 로그인 페이지 이동
- 계정 비활성화 시 → 로그인 차단, 안내 메시지 표시

## 4. 기본 LMS 기능

### 4.1 출결 관리

#### 기능 설명
강사가 수업별 출결을 입력하고, 운영자·수강생이 조회할 수 있다.

#### 요구사항

| ID | 요구사항 | 대상 | 우선순위 | Phase |
|----|----------|------|----------|-------|
| ATT-01 | 수업별 학생 목록 표시 `MUST` | TEACHER | 높음 | ★ 1 |
| ATT-02 | 출석 / 지각 / 조퇴 / 결석 4가지 상태 입력 `MUST` | TEACHER | 높음 | ★ 1 |
| ATT-03 | 전체 출석 버튼 (일괄 출석 처리) `SHOULD` | TEACHER | 중간 | ★ 1 |
| ATT-04 | 요일별 / 주별 / 전체 뷰 전환 `MUST` | TEACHER, ADMIN | 높음 | ★ 1 |
| ATT-05 | 날짜 이동 (이전·다음 날) `MUST` | TEACHER, ADMIN | 높음 | ★ 1 |
| ATT-06 | 결석 발생 시 학부모 문자 알림 자동 발송 `SHOULD` | 시스템 | 중간 | 2 |
| ATT-07 | 수강생 본인 출결 현황 조회 `MUST` | STUDENT | 높음 | ★ 1 |
| ATT-08 | 출결 통계 (출석률, 결석 횟수) 조회 `SHOULD` | ADMIN | 중간 | 2 |
| ATT-09 | 연속 결석 N회 시 운영자 알림 `SHOULD` | 시스템 | 중간 | 2 |

#### 데이터 구조

```
Attendance {
  id           String
  studentId    String
  classId      String       // 수업 (반 + 시간)
  date         Date
  status       Enum(PRESENT, LATE, EARLY_LEAVE, ABSENT)
  note         String?      // 사유
  createdBy    String       // 입력한 강사 ID
  createdAt    DateTime
}
```

#### 예외 처리

- 동일 학생·동일 수업·동일 날짜 중복 입력 시 → 덮어쓰기 (마지막 입력 기준)
- 과거 날짜 출결 수정 시 → ADMIN만 가능, 수정 이력 기록
- 수업이 없는 날 출결 입력 시 → 오류 메시지

| HW-03 | 미제출 학생 목록 조회 `MUST` | TEACHER | 높음 | ★ 1 |
| HW-04 | 마감일 당일 미제출 시 수강생 알림 `SHOULD` | 시스템 | 중간 | 2 |

| HW-06 | 파일 첨부 제출 (PDF, 이미지, 코드 파일) `SHOULD` | STUDENT | 중간 | 2 |

#### 데이터 구조

```
Assignment {
  id          String
  classId     String
  title       String
  description String?
  dueDate     DateTime
  createdBy   String
  createdAt   DateTime
}

Submission {
  id            String
  assignmentId  String
  studentId     String
  status        Enum(SUBMITTED, NOT_SUBMITTED)
  fileUrl       String?
  feedback      String?
  submittedAt   DateTime?
}
```

#### 예외 처리

- 마감일 이후 제출 → 지각 제출로 표시, 강사에게 알림
- 파일 용량 초과 (10MB 이상) → 업로드 차단, 안내 메시지

## 5. AI 핵심 기능

### 5.1 실시간 수업 코파일럿 ★

#### 기능 설명
수업 중 학생 질문을 입력·음성 인식하면 AI가 즉시 난이도별 답변을 생성해 강사에게 제공한다.

#### 요구사항

| ID | 요구사항 | 대상 | 우선순위 | Phase |
|----|----------|------|----------|-------|
| COP-01 | 수업 맥락 설정 (과목, 오늘 주제, 반 레벨 구성) `MUST` | TEACHER | 높음 | ★ 1 |
| COP-02 | 텍스트 질문 입력 `MUST` | TEACHER | 높음 | ★ 1 |
| COP-03 | 음성 인식 (Web Speech API) → 텍스트 변환 `SHOULD` | TEACHER | 중간 | ★ 1 |
| COP-04 | AI 응답 4개 카드 동시 생성 `MUST` | 시스템 | 높음 | ★ 1 |
| &nbsp;&nbsp;└ COP-04a | 초보자용 쉬운 설명 | | | |
| &nbsp;&nbsp;└ COP-04b | 예시 코드 (코드 과목) | | | |
| &nbsp;&nbsp;└ COP-04c | 심화 학생용 추가 질문 | | | |
| &nbsp;&nbsp;└ COP-04d | 판서용 핵심 정리 3줄 | | | |
| COP-05 | 첫 응답 3초 내 스트리밍 시작 `MUST` | 시스템 | 높음 | ★ 1 |
| COP-06 | 카드별 복사 버튼 `MUST` | TEACHER | 높음 | ★ 1 |
| COP-07 | "사용함" 체크 → 질문 이력 저장 `SHOULD` | TEACHER | 중간 | ★ 1 |
| COP-08 | 수업별 질문 히스토리 누적 조회 `SHOULD` | TEACHER | 중간 | 2 |

#### AI 프롬프트 입력 구성

```
- 수업 과목
- 오늘 주제
- 반 레벨 구성 (초보 N% / 기초 N% / 심화 N%)
- 학생 질문 텍스트
```

#### 예외 처리

- API 응답 지연 (5초 초과) → 로딩 상태 유지 + 재시도 버튼 표시
- API 오류 → 에러 메시지 표시, 질문 텍스트 보존
- 음성 인식 미지원 브라우저 → 텍스트 입력만 활성화

| REC-03 | AI 자동 정리 항목 생성 `MUST` | 시스템 | 높음 | 2 |
| &nbsp;&nbsp;└ REC-03a | 오늘 다룬 개념·범위 요약 | | | |
| &nbsp;&nbsp;└ REC-03b | 학생 질문 목록 추출 | | | |
| &nbsp;&nbsp;└ REC-03c | 강사 답변 요약 | | | |
| &nbsp;&nbsp;└ REC-03d | 다음 수업 이어갈 포인트 | | | |
| REC-04 | 정리 결과 수동 편집 가능 `SHOULD` | TEACHER | 중간 | 2 |
| REC-05 | 수업별 정리 이력 누적·검색 `SHOULD` | TEACHER | 중간 | 2 |
| REC-06 | 정리 내용 수강생 복습 자료로 자동 연동 `NICE` | 시스템 | 낮음 | 3 |

#### 예외 처리

- 파일 크기 초과 (100MB 이상) → 업로드 차단
