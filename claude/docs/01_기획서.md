**프로젝트명:** AcadeMind (가칭)
**작성일:** 2026.04.07
**버전:** v1.0

---

## 1. 프로젝트 개요

### 1.1 한 줄 정의

> 강사·수강생·운영자 모두가 쓰는 학원 관리 플랫폼에 AI를 더해, 수업 준비·학생 이탈·반복 업무 문제를 한 번에 해결한다.

### 1.2 배경

교육 현장에서 AI 활용 논의가 활발하지만, 대부분의 솔루션은 **학습 콘텐츠 제공**에 집중되어 있다.
실제 학원 운영 현장에서 강사·운영자가 겪는 문제는 다르다.

- 강사는 수업 외에도 학부모 응대, 반복 질문 처리, 수업 준비에 시간을 과도하게 쏟는다
- 운영자는 학생이 그만두고 나서야 이탈을 인지한다
- 수강생은 수업 후 뭘 복습해야 할지 모른 채 다음 수업을 맞는다

기존 학원 관리 시스템(출결·수납 중심)은 이 문제를 해결하지 못한다.
본 서비스는 **학원 운영의 기본 기능 위에 AI를 올려**, 세 역할 모두의 페인포인트를 동시에 해결한다.

### 1.3 목표

| 목표 | 지표 |
|------|------|
| 강사 수업 외 반복 업무 감소 | 질문 응대·수업 정리 시간 50% 단축 |
| 수강생 중도이탈 감소 | 이탈 징후 조기 감지로 이탈률 30% 개선 |
| 운영자 행정 업무 자동화 | 보강 조율·학부모 소통 자동화 |

## 2. 사용자 및 페인포인트

### 2.1 사용자 구분

```
학원 관리 플랫폼
├── 강사      → 수업 진행 + 학생 관리
├── 수강생    → 학습 참여 + 복습
└── 운영자    → 학원 전체 운영 관리
```

### 2.2 강사의 실제 문제

| 문제 | 현재 상황 |
|------|-----------|
| 반복 질문 응대 | 수업 중·후 동일 질문 수십 번 반복. 카톡 질문은 실시간 답변 불가 |
| 수업 준비 부담 | 난이도별 설명 자료·예시 코드 매번 새로 준비 |
| 수업 기록 없음 | 오늘 어디까지 했는지, 어떤 질문이 나왔는지 기록이 남지 않음 |
| 학부모 응대 | 감정적 민원에 매번 새로 답변 작성. 잘못 대응하면 이탈로 연결 |

### 2.3 수강생의 실제 문제

| 문제 | 현재 상황 |
|------|-----------|
| 복습 막막함 | 수업 후 뭘 복습해야 할지 모름. 다음날 내용 거의 휘발 |
| 질문 못 함 | 수업 중 모르는 게 생겨도 눈치 보여 손 못 듦 |
| AI 과제 의존 | AI로 과제 대신 해결. 강사 입장에서 과정 파악 불가 |

### 2.4 운영자의 실제 문제

| 문제 | 현재 상황 |
|------|-----------|
| 이탈 사후 인지 | 학생이 그만두겠다고 해야 알게 됨. 사전 신호 파악 불가 |
| 보강 조율 수작업 | 결석 시 강사·강의실·진도 맞는 반 직접 찾아야 함 |
| 분반 기준 불명확 | 시험 점수만으로 반 배정 → 수준 불일치 → 불만·이탈 |
| 이탈 원인 불명 | 왜 그만뒀는지 데이터 없이 감으로만 파악 |

### 3.2 AI 핵심 기능 상세

#### ① 실시간 수업 코파일럿 (강사) ★

**문제:** 수업 중 예상치 못한 질문에 즉각 대응하기 어렵고, 학생 수준이 제각각이라 같은 내용을 다양한 방식으로 설명해야 하는 부담이 크다.

**솔루션:**
- 학생 질문을 텍스트 입력 또는 음성으로 인식
- AI가 3초 내에 다음을 동시 생성
  - 초보자용 쉬운 설명 (비유·일상 예시)
  - 예시 코드 (코드 과목의 경우)
  - 심화 학생용 추가 질문
  - 판서용 핵심 정리 3줄
- 강사는 생성된 카드 중 골라서 바로 사용

**차별점:** 기존 AI는 학습 콘텐츠 생성에 집중. 본 기능은 수업 진행 자체를 실시간 보조.

#### ② 수업 녹음 → 자동 정리 (강사)

**문제:** 수업 후 오늘 어디까지 했는지, 어떤 질문이 나왔는지 기록이 남지 않는다.

**솔루션:**
- 수업 중 녹음 또는 종료 후 음성 파일 업로드
- AI가 자동으로
  - 오늘 다룬 개념·범위 요약
  - 학생 질문 목록 추출
  - 강사 답변 요약
  - 다음 수업 이어갈 포인트 제안
- 수업 기록이 자동으로 누적·검색 가능

#### ③ 반복 질문 필터 & 질문봇 (강사) ★

**문제:** 수업 외 시간 카톡·메시지로 동일한 질문이 반복 유입. 강사가 24시간 응대하는 구조.

**솔루션:**
- 학생이 플랫폼 내 질문 채널에 질문 입력
- AI가 수업 맥락·커리큘럼 기반으로 1차 자동 응대
- 자주 묻는 질문은 FAQ로 자동 축적
- AI가 해결 못한 질문만 강사에게 알림 전달

#### ④ 수업 후 복습 요약 발송 (수강생)

**문제:** 수업 후 뭘 복습해야 할지 모름. 다음날 내용 대부분 휘발.

**솔루션:**
- 수업 종료 후 AI가 자동으로
  - 오늘 배운 핵심 개념 3줄 요약
  - 개인 오답·질문 기반 맞춤 복습 퀴즈 3문제
  - 다음 수업 예습 포인트
- 앱 푸시 또는 문자로 발송

#### ⑤ 과제 과정 증빙 (수강생) ★

**문제:** AI로 과제를 대신 하는 학생 증가. 강사가 과정을 파악할 방법이 없음.

**솔루션:**
- 과제 제출 시 작성 이력·수정 시점 자동 기록
- AI 도움 사용 여부 셀프 체크 및 근거 첨부
- 강사에게 과정 포트폴리오 형태로 제출
- 결과물이 아닌 사고 과정 중심 평가 가능

#### ⑥ 이탈 예측 레이더 (운영자) ★

**문제:** 학생이 그만두겠다고 말할 때까지 이탈 신호를 모름.

**솔루션:**
- 다음 데이터를 복합 분석
  - 출결 패턴 (결석 빈도·연속 결석)
  - 과제 제출률 변화
  - 플랫폼 접속 빈도
  - 질문 빈도 감소
- 이탈 위험 학생을 점수+이유 함께 대시보드에 표시
- 운영자에게 "지금 연락 필요" 알림 발송

#### ⑦ 이탈 사유 해석 AI (운영자)

**문제:** 왜 그만뒀는지 데이터 없이 감으로만 파악.

**솔루션:**
- 이탈 학생의 출결·과제·상담·반배정 이력 분석
- 이탈 유형 자동 분류
  - 학습 난이도 불일치
  - 동기 저하 (출석률·과제 제출 하락)
  - 환경 요인 (결석 패턴)
  - 강사 관련 신호
- 유형별 재등록 접근 전략 제안

## 4. 차별점 및 경쟁 비교

### 4.1 기존 서비스와의 차이

| 구분 | 기존 학원 관리 앱 | 기존 AI 교육 솔루션 | AcadeMind |
|------|------------------|-------------------|-----------|
| 출결·수납 관리 | ✅ | ❌ | ✅ |
| AI 학습 콘텐츠 | ❌ | ✅ | 일부 |
| 실시간 수업 보조 | ❌ | ❌ | ✅ |
| 이탈 예측 | ❌ | ❌ | ✅ |
| 수업 기록 자동화 | ❌ | ❌ | ✅ |
| 강사·운영자·학생 통합 | 일부 | ❌ | ✅ |

### 4.2 핵심 차별점 요약

1. **수업 콘텐츠가 아닌 수업 진행 자체를 보조하는 AI**
   - 기존: 문제 생성, 개념 설명 콘텐츠
   - 본 서비스: 수업 중 실시간 강사 보조, 수업 후 자동 기록

2. **이탈을 사전에 막는 구조**
   - 기존: 이탈 후 재등록 마케팅
   - 본 서비스: 이탈 징후 감지 → 조기 개입 → 이탈 방지

3. **세 역할 모두의 문제를 하나의 플랫폼에서**
   - 강사·수강생·운영자가 같은 데이터를 공유하며 각자의 뷰에서 활용

## 5. 기대효과

### 5.1 강사

- 수업 외 반복 응대 시간 감소 → 수업 품질 집중
- 수업 기록이 자동 누적 → 다음 기수 재활용 가능
- 학부모 응대 부담 감소

### 5.2 수강생

- 수업 후 복습 방향 명확화 → 학습 연속성 향상
- 과제 과정 투명화 → AI 의존 줄이고 실력 중심 학습

### 5.3 운영자

- 이탈 조기 감지 → 매출 손실 예방
- 보강·분반 조율 자동화 → 행정 시간 절감
- 이탈 원인 데이터화 → 운영 전략 개선 근거 확보

## 6. 기술 스택 (개발 계획)

### 6.1 전체 구조

```
Frontend + Backend
└── Next.js 14 (App Router)
    ├── Server Components (데이터 패칭)
    ├── Server Actions (폼·mutation)
    └── Route Handlers (API 엔드포인트)

Database
└── PostgreSQL
    └── Prisma ORM

AI
├── Claude API (Anthropic) — 수업 코파일럿, 이탈 분석, 질문봇
└── Whisper API (OpenAI) — 수업 녹음 음성 인식

Auth
└── NextAuth.js (역할별: 강사 / 수강생 / 운영자)

Storage
└── Supabase Storage (수업 녹음 파일, 과제 첨부)

Deployment
└── Vercel
```

### 6.2 역할별 접근 권한

| 기능 | 강사 | 수강생 | 운영자 |
|------|------|--------|--------|
| 출결 입력 | ✅ | 본인 조회 | ✅ 전체 |
| 과제 관리 | ✅ | 본인 제출 | 조회 |
| 수업 코파일럿 | ✅ | ❌ | ❌ |
| 복습 요약 | ❌ | ✅ | ❌ |
| 이탈 레이더 | 담당반 | ❌ | ✅ 전체 |
| 정산·수강료 | ❌ | 본인 조회 | ✅ |
