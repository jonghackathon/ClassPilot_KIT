# 와이어프레임 — 수강생 화면 (STUDENT)

**연관 문서:** 02_권한_매트릭스_및_라우팅.md, 04_와이어프레임_공통_컴포넌트.md
**대상:** 프론트엔드
**설계 철학:** Mobile-First, Simple, Card-Based UI
**최대 너비:** `max-width: 640px` (모바일 우선, 중앙 정렬)
**하단 탭:** 홈 · 출결 · 과제 · 복습 — 모든 화면에 항상 표시

> 수강생 화면은 **모바일 퍼스트**로 설계합니다.
> 복잡한 표나 사이드바 없이, **카드 기반 UI**로 핵심 정보만 한 눈에 보여줍니다.
> 바텀 탭 내비게이션을 통해 주요 기능에 빠르게 접근할 수 있습니다.

---

## 공통 — 바텀 탭 바

```
┌──────────────────────────┐
│  🏠      📋      📝      📚  │
│  홈     출결    과제    복습  │
└──────────────────────────┘
```

- 현재 활성 탭: `Primary(#3B82F6)` + bold
- 비활성 탭: `gray-400`
- 높이: 56px, 안전 영역(Safe Area) 하단 패딩 포함
- 모든 수강생 화면 하단에 고정(fixed bottom)

### 4.2 인터랙션 설명

| 요소 | 동작 |
|------|------|
| `← 과제 목록` | `/student/assignments`로 뒤로가기 (변경사항 있으면 임시저장 확인) |
| 텍스트 에디터 | autosize(내용에 따라 높이 자동 조절), placeholder "여기에 답변을 작성하세요..." |
| 글자수 표시 | 실시간 글자수 카운트, 자동저장 상태("✅ 자동저장 완료" / "⏳ 저장 중...") |
| 자동저장 | **5분마다** 자동 임시저장 + 내용 변경 후 비활성 30초 시 저장 |
| AI 사용 여부 라디오 | "사용함" 선택 시 추가 텍스트 입력 필드 표시 (슬라이드 다운 애니메이션) |
| `[임시저장]` 버튼 | 수동 임시저장, 토스트 알림 "임시저장 완료" |
| `[제출하기]` 버튼 | 확인 모달 표시 → 확인 시 제출 API 호출 → 성공 시 과제 목록으로 이동 |
| 제출 확인 모달 | "과제를 제출할까요? 제출 후에는 수정할 수 없습니다." |

### 4.3 데이터 연결

| API | 설명 |
|-----|------|
| `GET /api/assignments/[id]` | 과제 상세 정보 (과제명, 마감일, 설명) |
| `GET /api/assignments/[id]/draft` | 임시저장 데이터 불러오기 |
| `PUT /api/assignments/[id]/draft` | 임시저장 (5분 자동 + 수동) |
| `POST /api/assignments/[id]/submit` | 과제 제출 (답변 내용, AI 사용 여부, AI 사용 설명) |
