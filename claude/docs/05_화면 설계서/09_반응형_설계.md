## 1. 브레이크포인트 정의

| 이름 | 범위 | Tailwind | 주 사용자 | 레이아웃 특성 |
|------|------|----------|---------|-------------|
| **모바일** | ~767px | 기본 (no prefix) | 수강생, 강사(이동 중) | 1컬럼, 바텀 탭/네비 |
| **태블릿** | 768~1023px | `md:` | 강사(수업 중) | 2컬럼 가능, 탭 축약 |
| **데스크탑** | 1024~1279px | `lg:` | 강사(기본) | 풀 탭 네비, 2~3컬럼 |
| **와이드** | 1280px~ | `xl:` | 운영자 | 사이드바+넓은 콘텐츠, 4컬럼 |

## 3. 역할별 반응형 상세

### 3.1 운영자 — 데스크탑 우선

운영자는 주로 데스크탑에서 사용하지만, 긴급 확인 시 모바일에서도 접근합니다.

**데스크탑 (1024px+):**
```
┌──────────┬──────────────────────────────────────────┐
│  사이드바  │  탑바                                      │
│  (240px)  ├──────────────────────────────────────────┤
│          │                                          │
│          │  콘텐츠 (max-w: 1440px, 양쪽 패딩 24px)    │
│          │                                          │
└──────────┴──────────────────────────────────────────┘
```

**모바일 (~1023px):**
```
┌──────────────────────────┐
│  [≡] 로고          [🔔][👤]│
├──────────────────────────┤
│                          │
│  콘텐츠 (패딩 16px)       │
│                          │
└──────────────────────────┘
```

**핵심 변환:**
- 사이드바 → 햄버거 오버레이
- 4열 StatCard → 2열
- DataTable → 카드 리스트 (주요 정보만 표시)
- 시간표 그리드 → 일별 리스트
- 학생 상세 좌우 분할 → 상하 스택

### 3.2 강사 — 태블릿 중심

강사는 수업 중 태블릿, 외부에서 모바일, 사무실에서 데스크탑 사용.

**데스크탑 (1024px+):**
```
┌─────────────────────────────────────────┐
│  탑바                                     │
├─────────────────────────────────────────┤
│  홈  │  출결  │  과제  │  진도  │ AI코파일럿│
├─────────────────────────────────────────┤
│  콘텐츠 (max-w: 1200px)                   │
└─────────────────────────────────────────┘
```

**태블릿 (768~1023px):**
```
┌──────────────────────────────┐
│  탑바                          │
├──────────────────────────────┤
│  홈 │ 출결 │ 과제 │ 진도 │ AI │  ← 텍스트 축약
├──────────────────────────────┤
│  콘텐츠                        │
└──────────────────────────────┘
```

**모바일 (~767px):**
```
┌──────────────────────────┐
│  탑바                      │
├──────────────────────────┤
│  콘텐츠                    │
├──────────────────────────┤
│ 🏠 │ 📋 │ 📝 │ 📈 │ ✨  │  ← 하단 아이콘 탭
└──────────────────────────┘
```

**핵심 변환:**
- 탭 네비 → 하단 5아이콘 탭 (✨더보기 → 녹음/질문봇/이탈)
- 출결 상태 버튼: 48px 터치 타겟 (모바일)
- 코파일럿 4카드: 2x2 → 1열 스택
- 과제 상세 테이블 → 카드 리스트

### 3.3 수강생 — 모바일 우선

수강생은 주로 스마트폰에서 사용. 데스크탑은 보조.

**모바일 (기본):**
```
┌──────────────────────────┐
│  미니 탑바 (48px)          │
├──────────────────────────┤
│                          │
│  콘텐츠 (max-w: 640px)    │
│  패딩: 16px              │
│                          │
├──────────────────────────┤
│  바텀 탭 (64px)            │
└──────────────────────────┘
```

**데스크탑 (1024px+):**
```
┌─────────────────────────────────────────┐
│  로고  홈  출결  과제  복습        [🔔][👤]│  ← 바텀탭 → 상단 네비
├─────────────────────────────────────────┤
│              ┌────────────────┐         │
│              │  콘텐츠         │         │  ← max-w: 640px, 중앙 정렬
│              │  (모바일과 동일)  │         │
│              └────────────────┘         │
└─────────────────────────────────────────┘
```

**핵심 변환:**
- 바텀 탭 → 상단 네비 (1024px+)
- 콘텐츠 max-width 640px 유지 (데스크탑에서도)
- 과제 에디터: 모바일은 전체 높이, 데스크탑은 고정 높이
- 복습 퀴즈: 선택지 버튼 크기 유지 (48px)

## 4. DataTable → 카드 리스트 변환

### 4.1 변환 규칙

768px 이하에서 DataTable의 행이 카드로 변환됩니다.

**원본 테이블 (데스크탑):**
```
│ #  │  이름  │ 학년 │  수강 반 │ 출석률 │ 이탈위험 │ 액션  │
│ 1  │ 김민준 │ 초4  │ (화-초4) │  80%  │ 🔴 위험 │ [상세]│
│ 2  │ 이수현 │ 초6  │ (목-초6) │  95%  │ 🟢 정상 │ [상세]│
```

**변환된 카드 (모바일):**
```
┌──────────────────────────┐
│  김민준  초등 4학년         │  ← 이름 + 학년 (1줄)
│  (화-초4) 친절한 설명       │  ← 반 이름 (2줄)
│  출석률 80%   🔴 위험      │  ← 핵심 지표 + 상태 (3줄)
│                   [→]     │  ← 상세 진입 화살표
├──────────────────────────┤
│  이수현  초등 6학년         │
│  (목-초6) 고전1300         │
│  출석률 95%   🟢 정상      │
│                   [→]     │
└──────────────────────────┘
```

### 4.2 카드에 표시하는 정보 우선순위

모든 컬럼을 카드에 넣을 수 없으므로, 핵심 정보만 선택:

| 화면 | 카드에 표시 | 카드에서 제외 (상세에서 확인) |
|------|-----------|--------------------------|
| 학생 목록 | 이름, 학년, 반, 출석률, 위험도 | 이메일 |
| 강사 목록 | 이름, 담당 반, 수업 수 | 이메일 |
| 수강료 | 학생명, 반, 금액, 상태 | 납부일 |
| 과제 제출 현황 | 학생명, 상태, AI사용, 이력 수 | 제출 시간 |

## 7. 터치 인터랙션 (모바일)

| 제스처 | 적용 화면 | 동작 |
|-------|---------|------|
| **탭** | 출결 상태 버튼 | 즉시 상태 변경 (낙관적 업데이트) |
| **풀 다운** | 모든 목록 페이지 | 새로고침 (SWR revalidate) |
| **스와이프 좌우** | 출결 달력 | 월 이동 |
| **스와이프 좌우** | 시간표 요일 탭 | 요일 전환 |
| **길게 누르기** | - | 미사용 (오동작 방지) |

## 8. safe-area 대응

iPhone 노치·홈 바 영역 대응:

```css
/* 바텀 탭 */
.bottom-tab {
  padding-bottom: env(safe-area-inset-bottom);
}

/* 모바일 바텀시트 모달 */
.bottom-sheet {
  padding-bottom: env(safe-area-inset-bottom);
}
```

## 9. 성능 고려사항

| 항목 | 전략 |
|------|------|
| 이미지 최적화 | Next.js `<Image>` 컴포넌트 (자동 리사이징, WebP) |
