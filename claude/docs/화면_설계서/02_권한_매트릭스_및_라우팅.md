## 2. 데이터 범위 권한

같은 API를 호출하더라도 역할에 따라 **볼 수 있는 데이터 범위**가 다릅니다.

| 데이터 종류 | ADMIN | TEACHER | STUDENT |
|------------|-------|---------|---------|
| **출결 기록** | 전체 학생 (CRUD + 수정 감사 로그) | 담당 반 학생만 (Create/Read/Update) | 본인만 (Read) |
| **과제** | 전체 조회 (Read) | 담당 반 (Create/Read/Update/Delete) | 본인 수강 반 (Read + 제출) |
| **학생 정보** | 전체 (CRUD) | 담당 반 학생 (Read) | 본인만 (Read) |
| **강사 정보** | 전체 (CRUD) | 본인만 (Read/Update) | ❌ |
| **반 정보** | 전체 (CRUD) | 담당 반 (Read) | 수강 반 (Read) |
| **결제 정보** | 전체 (CRUD) | ❌ | ❌ |
| **이탈 예측** | 전체 학생 (Read) | 담당 반 학생 (Read) | ❌ |
| **AI 코파일럿** | ❌ | 본인 세션 (CRUD) | ❌ |
| **복습 요약** | ❌ | 담당 반 (Read) | 본인만 (Read) |
| **질문/FAQ** | 전체 (Read) | 담당 반 (CRUD + 답변) | 본인 질문 (Create/Read) |
| **녹음 요약** | ❌ | 본인 녹음 (CRUD) | ❌ |
| **민원** | 전체 (CRUD) | ❌ | ❌ |
| **상담 기록** | 전체 (CRUD) | 담당 학생 (Read) | ❌ |

## 4. 미들웨어 라우팅 가드

### 4.1 라우팅 로직

```
middleware.ts 실행 순서:

1. 세션 확인
   ├── 세션 없음 + 보호된 경로 → /login 리다이렉트
   └── 세션 있음 → 2번으로

2. 로그인 페이지 접근 시
   ├── 이미 로그인 → 역할별 홈으로 리다이렉트
   └── 미로그인 → 로그인 페이지 표시

3. 역할별 경로 확인
   ├── ADMIN  + /admin/*  접근 → 통과 ✅
   ├── TEACHER + /teacher/* 접근 → 통과 ✅
   ├── STUDENT + /student/* 접근 → 통과 ✅
   └── 역할 불일치 → 해당 역할 홈으로 리다이렉트

4. 역할별 홈 경로
   ├── ADMIN   → /admin/dashboard
   ├── TEACHER → /teacher/dashboard
   └── STUDENT → /student/home
```

### 4.2 리다이렉트 규칙

| 상황 | 현재 경로 | 리다이렉트 대상 |
|------|---------|---------------|
| 비로그인 + 보호 경로 | `/admin/dashboard` | `/login?callbackUrl=/admin/dashboard` |
| 로그인됨 + 로그인 페이지 | `/login` | 역할별 홈 |
| ADMIN + 강사 경로 | `/teacher/attendance` | `/admin/dashboard` |
| TEACHER + 운영자 경로 | `/admin/students` | `/teacher/dashboard` |
| STUDENT + 강사 경로 | `/teacher/copilot` | `/student/home` |
| 루트 경로 | `/` | `/login` (비로그인) 또는 역할별 홈 |

### 4.3 callbackUrl 처리

로그인 후 원래 접근하려던 페이지로 리다이렉트:

```
1. 비로그인 사용자가 /admin/students 접근
2. → /login?callbackUrl=%2Fadmin%2Fstudents 로 리다이렉트
3. 로그인 성공
4. → callbackUrl이 해당 역할의 허용 경로인지 검증
5. → 허용되면 /admin/students로, 아니면 역할별 홈으로
```

## 5. 보호 경로 설정

### 5.1 공개 경로 (인증 불필요)

```
/login
/api/auth/*     (NextAuth 인증 관련 엔드포인트)
/_next/*        (Next.js 정적 자원)
/favicon.ico
```

### 5.2 API 라우트 권한

API 라우트에서도 동일한 권한 검사 적용:

```typescript
// 예시: 출결 조회 API
GET /api/attendance

ADMIN   → 전체 학생 출결 반환 (필터 가능)
TEACHER → teacherId 기준 담당 반 학생만 반환
STUDENT → 본인 studentId 출결만 반환
비로그인  → 401 Unauthorized
```

### 5.3 동적 라우트 소유권 검증

`[id]` 같은 동적 라우트에서 해당 리소스에 대한 접근 권한 추가 검증:

| 경로 | 검증 내용 |
|------|---------|
| `/teacher/assignments/[id]` | 해당 과제가 강사 담당 반의 과제인지 |
| `/teacher/copilot/[lessonId]` | 해당 수업이 강사 담당 수업인지 |
| `/student/assignments/[id]` | 해당 과제가 학생 수강 반의 과제인지 |
| `/student/review/[id]` | 해당 복습 자료가 학생 본인의 것인지 |
| `/admin/students/[id]` | ADMIN은 전체 접근 가능 (추가 검증 불필요) |
