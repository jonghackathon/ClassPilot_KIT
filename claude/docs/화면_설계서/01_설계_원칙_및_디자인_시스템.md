## 1. 역할별 설계 철학

AcadeMind는 세 역할(수강생·강사·운영자)이 **완전히 다른 맥락**에서 사용합니다.
각 역할의 사용 환경·목적·인지 부하 수준이 다르므로, 화면 설계 기준도 분리합니다.

| 역할 | 핵심 원칙 | UI 밀도 | 주 사용 환경 | 네비게이션 |
|------|----------|---------|------------|-----------|
| **수강생** | 심플. 내 것만 보임. 탭 4개 이내. 버튼 크게. 최소 인지 부하. | 낮음 (카드 중심) | 모바일 (스마트폰) | 하단 탭 바 |
| **강사** | 수업 흐름 중심. 출결→과제→AI 기능이 1~2 탭 내. 빠른 전환. | 중간 (리스트+카드) | 태블릿 / 데스크탑 | 상단 탭 네비 |
| **운영자** | 데이터 중심. 한 화면에 핵심 지표. 필터·검색·정렬 필수. | 높음 (테이블+대시보드) | 데스크탑 | 좌측 사이드바 |

### 설계 원칙 상세

#### 수강생 — "3초 안에 할 일을 파악"
- 홈 화면에서 **오늘 수업 / 미제출 과제 / 새 복습자료** 즉시 확인
- 내 데이터만 표시 (다른 학생 정보 절대 노출 안 됨)
- 한 화면에 하나의 주요 액션 (과제 작성, 복습 확인 등)
- 큰 터치 타겟 (최소 48px), 명확한 CTA 버튼
- 스크롤 최소화 — 핵심 정보가 첫 화면에

#### 강사 — "수업 전·중·후 흐름을 끊지 않는다"
- 수업 전: 출결 입력 → 수업 중: AI 코파일럿 → 수업 후: 녹음 정리
- 이 흐름이 1~2번의 탭 전환으로 완성되어야 함
- 반복 작업 최소화 (전체출석, 일괄 처리 등)
- 수업 중 사용 → 태블릿에서 한 손 조작 가능해야 함

#### 운영자 — "한눈에 전체, 클릭하면 상세"
- 대시보드에서 KPI 4개 + 위험 알림 + 오늘 일정 한 번에
- 모든 목록에 검색·필터·정렬 필수
- 드릴다운: 요약 → 목록 → 상세 (3단계 정보 계층)
- 데이터 테이블 중심, 대량 데이터 처리 가능

## 2. 반응형 브레이크포인트

```
모바일   :     ~768px   →  수강생·강사 주 사용 환경 (1컬럼)
태블릿   :  768~1024px  →  강사 수업 중 사용 (2컬럼 가능)
데스크탑  : 1024~1280px  →  강사 기본 환경 (탭 네비 풀 표시)
와이드    : 1280px~      →  운영자 주 사용 환경 (사이드바 + 넓은 콘텐츠)
```

Tailwind CSS 매핑:
```
sm: 640px    (모바일 큰 화면)
md: 768px    (태블릿 시작)
lg: 1024px   (데스크탑 시작)
xl: 1280px   (와이드 시작)
2xl: 1536px  (대형 모니터)
```

## 3. 컬러 시스템

### 3.0 컬러 설계 근거

> **레퍼런스 리서치 결과 (2025~2026 교육 플랫폼 트렌드)**
>
> - **파란색(Blue/Indigo)** → 신뢰·집중력 향상. 교육 분야에서 가장 선호되는 Primary 컬러
> - **녹색(Green)** → 성장·균형·차분함. 긍정적 상태(출석·완료) 표현에 최적
> - **인디고/보라(Indigo)** → 호기심·탐구심 유발. AI 기능 강조에 적합
> - **빨간색 과다 사용은 스트레스 유발** → 상태 표시(결석·위험)에만 제한적으로 사용
> - **너무 무채색이면 학습 동기 저하** → 충분한 컬러 액센트 필수
> - **소프트 파스텔 + 비비드 액센트** 조합이 현대 교육 플랫폼의 주류
>
> **결정:** Primary를 `sky-500`(하늘색)에서 **`indigo-600`(깊은 인디고)**로 변경.
> 이유: 전문성+신뢰감+AI 기능과의 시각적 통일성. shadcn/ui 기본 테마와도 호환.

### 3.5 상태별 컬러 매핑

```
출결:
  PRESENT     → bg-green-100  text-green-700  border-green-200
  LATE        → bg-amber-100  text-amber-700  border-amber-200
  EARLY_LEAVE → bg-orange-100 text-orange-700 border-orange-200
  ABSENT      → bg-red-100    text-red-700    border-red-200

과제:
  SUBMITTED     → bg-green-100  text-green-700
  NOT_SUBMITTED → bg-red-100    text-red-700
  LATE_SUBMIT   → bg-amber-100  text-amber-700

이탈 위험:
  SAFE    → bg-green-100  text-green-700
  WARNING → bg-amber-100  text-amber-700
  DANGER  → bg-red-100    text-red-700

납부:
  PAID    → bg-green-100  text-green-700
  UNPAID  → bg-red-100    text-red-700
  PARTIAL → bg-amber-100  text-amber-700

AI 기능:
  코파일럿 세션  → bg-violet-50  border-l-4 border-violet-500
  AI 응답 카드  → bg-violet-50  text-violet-900
  AI 생성 중    → bg-indigo-50  animate-pulse
  AI 배지       → bg-violet-100  text-violet-700
```

### 3.6 그래디언트 (특수 용도)

```
로그인 페이지 배경 (상단):
  from-indigo-600 via-violet-600 to-purple-600
  → 브랜드 아이덴티티 강조. 로그인 폼은 White 카드로 대비.

코파일럿 헤더:
  from-violet-600 to-indigo-600
  → AI 기능의 특별감. 텍스트 white.

이탈 위험도 바:
  0~29: green-500
  30~59: from-green-500 via-amber-500
  60~100: from-amber-500 via-red-500
  → 숫자에 따라 색상이 자연스럽게 전이

대시보드 StatCard 호버:
  shadow-sm → shadow-md + 미세 scale(1.02)
  → 인터랙티브 느낌
```

## 7. 버튼 시스템

### 7.1 버튼 변형

| 변형 | 배경 | 텍스트 | 테두리 | 용도 | 예시 Tailwind |
|------|------|-------|--------|------|-------------|
| **Primary** | `indigo-600` | `white` | 없음 | 주요 액션 (로그인, 등록, 제출) | `bg-indigo-600 hover:bg-indigo-700 text-white` |
| **Secondary** | `white` | `slate-700` | `slate-300` | 보조 액션 (취소, 필터, 뒤로) | `bg-white hover:bg-slate-50 text-slate-700 border border-slate-300` |
| **Danger** | `red-600` | `white` | 없음 | 위험 액션 (삭제, 이탈 처리) | `bg-red-600 hover:bg-red-700 text-white` |
| **Outline** | 투명 | `indigo-600` | `indigo-300` | 보조 Primary (수강 등록 등) | `bg-transparent hover:bg-indigo-50 text-indigo-600 border border-indigo-300` |
| **Ghost** | 투명 | `slate-600` | 없음 | 텍스트 버튼 (전체보기 →) | `hover:bg-slate-100 text-slate-600` |
| **Accent** | `violet-600` | `white` | 없음 | AI 관련 (코파일럿 시작) | `bg-violet-600 hover:bg-violet-700 text-white` |
| **Icon** | 투명 | `slate-500` | 없음 | 아이콘만 (알림, 닫기) | `hover:bg-slate-100 text-slate-500 rounded-full` |

```
버튼 시각 예시:

 ┌───────────────┐  ┌───────────────┐  ┌───────────────┐
 │    등록하기     │  │    취  소      │  │  🔴 이탈 처리   │
 │   (Primary)    │  │  (Secondary)   │  │   (Danger)     │
 └───────────────┘  └───────────────┘  └───────────────┘
  bg-indigo-600      bg-white/border     bg-red-600

 ┌───────────────┐  ┌───────────────┐  ┌───────────────┐
 │  수강 등록 →   │  │  전체보기 →    │  │ ✨ 코파일럿 시작│
 │   (Outline)    │  │   (Ghost)     │  │   (Accent)     │
 └───────────────┘  └───────────────┘  └───────────────┘
  border-indigo-300   text only           bg-violet-600
```

### 7.2 버튼 크기

| 크기 | 높이 | 패딩 | 폰트 | 둥글기 | 용도 |
|------|------|------|------|-------|------|
| **xs** | 28px | 4px 8px | 12px | rounded-md | 배지 내 액션, 태그 |
| **sm** | 32px | 6px 12px | 13px | rounded-md | 테이블 내 액션 |
| **md** | 40px | 8px 16px | 14px | rounded-lg | 기본 (데스크탑 전역) |
| **lg** | 48px | 12px 24px | 16px | rounded-xl | 수강생 모바일 CTA, 로그인 |

### 7.3 버튼 상태

```
기본 (Default):
  ┌───────────────┐
  │    등록하기     │    bg-indigo-600, text-white
  └───────────────┘

호버 (Hover):
  ┌───────────────┐
  │    등록하기     │    bg-indigo-700 (1단계 어둡게)
  └───────────────┘    cursor: pointer

포커스 (Focus):
  ┏━━━━━━━━━━━━━━━┓
  ┃    등록하기     ┃    ring-2 ring-offset-2 ring-indigo-500
  ┗━━━━━━━━━━━━━━━┛    (외곽에 인디고 링)

비활성 (Disabled):
  ┌───────────────┐
  │    등록하기     │    opacity-50, cursor-not-allowed
  └───────────────┘    포인터 이벤트 없음

로딩 (Loading):
  ┌───────────────┐
  │  ⏳ 등록하기    │    스피너 애니메이션 + 텍스트 유지
  └───────────────┘    opacity-70, 클릭 비활성

활성/선택됨 (Active — 출결 버튼 등):
  ┌───────────────┐
  │  ✓ 출석       │    bg-green-600, text-white, ring-2 ring-green-300
  └───────────────┘    선택 해제 가능 (토글)
```

### 7.4 출결 전용 버튼 (특수)

출결 입력 시 사용하는 4상태 토글 버튼 세트:

```
미선택 상태 (기본):
  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
  │ 출석  │ │ 지각  │ │ 조퇴  │ │ 결석  │    모두 bg-white, border-slate-300
  └──────┘ └──────┘ └──────┘ └──────┘    text-slate-600

출석 선택:
  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
  │✓ 출석 │ │ 지각  │ │ 조퇴  │ │ 결석  │    출석: bg-green-600, text-white
  └──────┘ └──────┘ └──────┘ └──────┘    나머지: bg-white (비활성 느낌)

결석 선택:
  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
  │ 출석  │ │ 지각  │ │ 조퇴  │ │✓ 결석 │    결석: bg-red-600, text-white
  └──────┘ └──────┘ └──────┘ └──────┘

각 상태별 색상:
  출석 선택  → bg-green-600  text-white  ring-green-200
  지각 선택  → bg-amber-500  text-white  ring-amber-200
  조퇴 선택  → bg-orange-500 text-white  ring-orange-200
  결석 선택  → bg-red-600    text-white  ring-red-200

크기: sm (32px, 데스크탑) / md (40px, 모바일 — 터치 타겟 확보)
```

## 10. 접근성 기준

| 기준 | 규격 | 적용 |
|------|------|------|
| 색상 대비 | WCAG AA (4.5:1 이상) | 모든 텍스트·배경 조합 |
| 터치 타겟 | 최소 44×44px | 모바일 버튼·링크 |
| 키보드 탐색 | Tab / Shift+Tab / Enter / Escape | 모든 인터랙티브 요소 |
| 포커스 표시 | `ring-2 ring-offset-2 ring-indigo-500` | 포커스된 요소에 항상 표시 |
| 스크린 리더 | ARIA 레이블 | 아이콘 버튼, 상태 배지 |
| 색상 독립 | 아이콘+텍스트 병행 | 출결·이탈 상태 (색맹 대응) |
| 모션 | `prefers-reduced-motion` 대응 | 애니메이션 비활성화 옵션 |
| 대비 검증 | indigo-600 on white = 6.3:1 ✅ | Primary 텍스트 사용 가능 |
| | red-600 on red-100 = 4.7:1 ✅ | 에러 배지 가독성 확보 |
| | slate-500 on white = 4.6:1 ✅ | 보조 텍스트 최소 기준 충족 |
