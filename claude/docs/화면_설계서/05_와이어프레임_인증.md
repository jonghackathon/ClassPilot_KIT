### 1.3 상태별 화면

#### 초기 상태
- 이메일 필드에 자동 포커스
- 로그인 버튼 활성화 상태 (빈 입력에서도 클릭 가능 → 유효성 에러 표시)
- 비밀번호 필드 type="password"

#### 입력 중
- React Hook Form 실시간 유효성 검사
- 유효하지 않은 필드: 빨간 테두리 + 하단 에러 메시지

#### 로딩 중
```
┌───────────────────────────┐
│    ⏳  로그인 중...         │  ← 스피너 + 텍스트, 버튼 비활성
└───────────────────────────┘
  opacity: 0.7, cursor: not-allowed
```

#### 로그인 실패
```
┌───────────────────────────────────┐
│  ❌  이메일 또는 비밀번호가          │  ← 폼 상단 에러 배너
│     올바르지 않습니다               │     bg-red-50, text-red-700
└───────────────────────────────────┘
```

#### 로그인 성공
- 역할별 리다이렉트 (즉시, 별도 성공 화면 없음)

### 1.4 유효성 검사

```typescript
const loginSchema = z.object({
  email: z
    .string()
    .min(1, "이메일을 입력해주세요")
    .email("이메일 형식이 올바르지 않습니다"),
  password: z
    .string()
    .min(1, "비밀번호를 입력해주세요"),
})
```

| 필드 | 규칙 | 에러 메시지 |
|------|------|-----------|
| 이메일 | 필수 | "이메일을 입력해주세요" |
| 이메일 | 이메일 형식 | "이메일 형식이 올바르지 않습니다" |
| 비밀번호 | 필수 | "비밀번호를 입력해주세요" |

### 1.5 역할별 리다이렉트

| 역할 | 리다이렉트 대상 |
|------|---------------|
| ADMIN | `/admin/dashboard` |
| TEACHER | `/teacher/dashboard` |
| STUDENT | `/student/home` |

**callbackUrl 처리:**
```
/login?callbackUrl=%2Fadmin%2Fstudents

로그인 성공 후:
1. callbackUrl 파라미터 존재 확인
2. callbackUrl이 해당 역할의 허용 경로인지 검증
3. 허용 → callbackUrl로 이동
4. 불허 → 역할별 기본 홈으로 이동
```

### 1.6 에러 시나리오

| 시나리오 | 에러 코드 | 사용자에게 표시 |
|---------|---------|--------------|
| 잘못된 이메일/비밀번호 | 401 | "이메일 또는 비밀번호가 올바르지 않습니다" |
| 비활성화된 계정 | 403 | "비활성화된 계정입니다. 관리자에게 문의하세요" |
| 서버 오류 | 500 | "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요" |
| 네트워크 오류 | - | "네트워크 연결을 확인해주세요" |
| 너무 많은 시도 | 429 | "로그인 시도가 너무 많습니다. 5분 후 다시 시도해주세요" |

### 1.7 접근성

- 이메일 필드: `aria-label="이메일"`, `autocomplete="email"`
- 비밀번호 필드: `aria-label="비밀번호"`, `autocomplete="current-password"`
- 에러 메시지: `aria-live="polite"`, `role="alert"`
- 비밀번호 토글: `aria-label="비밀번호 표시"` / `"비밀번호 숨기기"`
- Enter 키 → 폼 제출
- Tab 순서: 이메일 → 비밀번호 → 비밀번호 토글 → 로그인 버튼
