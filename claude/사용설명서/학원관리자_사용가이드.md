## 1. 로그인

### 접속 경로
```
http://localhost:3000/login
```

### 로그인 흐름
1. 로그인 페이지에서 **"관리자 / 강사"** 버튼 선택
2. 이메일과 비밀번호 입력
3. 로그인 후 `/admin/dashboard` 로 자동 이동

### 테스트 계정
| 항목 | 값 |
|------|-----|
| 이메일 | `admin@academind.kr` |
| 비밀번호 | `1234` |
| 역할 | ADMIN |
| 이름 | 정태 |

## 2. 대시보드

**경로:** `/admin/dashboard`

학원 전체 현황을 한눈에 파악할 수 있는 메인 화면입니다.

### 제공 정보
- **오늘의 수업 현황**: 예정 수업 수, 완료 수업 수
- **전체 학생 수**: 등록 학생 수, 이번 달 신규 등록 수
- **출석률 통계**: 전체 평균 출석률
- **결제 현황**: 이번 달 수납액, 미납 건수
- **위험 학생 알림**: 이탈 위험도가 높은 학생 목록
- **최근 민원**: 미처리 민원 요약

### 사용 흐름
```
로그인 → 대시보드에서 전체 현황 파악
       → 위험 학생이 있으면 이탈예측 메뉴로 이동
       → 미처리 민원이 있으면 민원 관리 메뉴로 이동
```

## 3. 반 관리

**경로:** `/admin/classes`

학원의 모든 수업 반을 생성하고 관리합니다.

### 반 목록 조회
- 현재 운영 중인 모든 반 목록 확인
- 반별 담당 강사, 수강 학생 수 표시
- 반 상태(활성/비활성) 필터링

### 반 생성 흐름
1. 반 목록 페이지에서 **"반 추가"** 버튼 클릭
2. 아래 정보 입력:
   - 반 이름 (예: 논술 중급반)
   - 과목
   - 담당 강사 선택
   - 정원 설정
3. **"저장"** 버튼 클릭

### 반 상세 관리
**경로:** `/admin/classes/[반ID]`

- **담당 강사 배정**: 강사 추가/변경/삭제
- **수강생 관리**: 학생 추가/삭제
- **커리큘럼 연결**: 해당 반에 커리큘럼 적용
- **시간표 설정**: 수업 요일/시간 설정

### 현재 등록된 반 (테스트 데이터)
| 반 이름 | 과목 | 담당 강사 |
|---------|------|-----------|
| 논술 중급반 | 국어 논술 | 박강사 |
| 수학 A반 | 수학 | 김강사 |
| 영어 B반 | 영어 | 이강사 |
| 국어 A반 | 국어 | 정강사 |

## 4. 학생 관리

**경로:** `/admin/students`

### 학생 목록 조회
- 전체 학생 목록 (이름, 학교, 학년, 등록 반)
- 검색 및 필터링 (학교, 학년, 반별)
- 이탈 위험도 표시

### 학생 등록 흐름
1. 학생 목록 페이지에서 **"학생 추가"** 버튼 클릭
2. 아래 정보 입력:
   - 이름, 학교, 학년
   - 부모 연락처 (이름, 전화번호, 관계)
   - PIN 번호 설정 (4자리, 학생 로그인용)
   - 수강 반 선택
3. 학생 코드 자동 발급 (예: 2025-001)

### 학생 상세 관리
**경로:** `/admin/students/[학생ID]`

- **기본 정보**: 이름, 학교, 학년 수정
- **부모 연락처**: 추가/수정/삭제
- **수강 반**: 반 이동, 추가 등록
- **출석 현황**: 월별 출석 통계 확인
- **결제 내역**: 수강료 납부 이력
- **상담 기록**: 학부모 상담 이력
- **이탈 위험도**: 현재 위험도 수준 및 사유
- **PIN 초기화**: 학생 PIN 번호 재설정

### 현재 등록된 학생 (테스트 데이터)
| 이름 | 학생코드 | 학교 | 학년 | PIN | 부모 연락처 |
|------|----------|------|------|-----|-------------|
| 민수 | 2025-001 | 중앙중 | 중2 | 1234 | 010-9999-0001 |
| 이지은 | 2025-002 | 서초중 | 중2 | 1234 | - |
| 정우진 | 2025-003 | 반포중 | 중3 | 1234 | - |
| 한소영 | 2025-004 | 잠원중 | 중2 | 1234 | - |
| 김태호 | 2025-005 | 서일중 | 중1 | 1234 | - |

## 6. 시간표 관리

**경로:** `/admin/schedule`

### 시간표 설정 흐름
1. 시간표 관리 페이지 접속
2. **"수업 추가"** 버튼 클릭
3. 아래 정보 입력:
   - 수업 반 선택
   - 요일 및 시작/종료 시간
   - 반복 여부 (매주/격주)
4. 저장 시 해당 반 학생들의 수업 일정에 자동 반영

### 수업 생성 (개별 수업)
시간표와는 별개로 특정 날짜의 개별 수업을 생성할 수 있습니다:
- 보충 수업, 특강 등
- 강사 코파일럿 기능과 연동

## 7. 결제 관리

**경로:** `/admin/payments`

### 결제 현황 조회
- 전체 결제 내역 목록
- 상태별 필터: 납부완료 / 미납 / 부분납부
- 월별 수납 통계

### 결제 등록 흐름
1. 결제 관리 페이지에서 **"결제 등록"** 버튼 클릭
2. 아래 정보 입력:
   - 학생 선택
   - 수강료 금액
   - 납부일
   - 납부 방법 (현금/카드/계좌이체)
   - 납부 상태 (납부완료/미납/부분납부)
3. 저장

### 납부 상태 변경
- 미납 → 납부완료: 결제 상세에서 상태 변경
- 부분납부 기록 가능 (잔여금 자동 계산)

## 8. 민원 관리

**경로:** `/admin/complaints`

학부모나 학생의 민원을 접수하고 처리합니다.

### 민원 목록 조회
- 전체 민원 목록
- 상태별 필터: 접수 / 처리중 / 처리완료

### 민원 처리 흐름
1. 민원 목록에서 해당 민원 클릭
2. 민원 내용 확인
3. **"AI 답변 초안 생성"** 버튼으로 AI가 초안 작성 (선택)
4. 답변 내용 수정 후 **"답변 전송"**
5. 민원 상태가 자동으로 "처리완료"로 변경

### 민원 등록 (관리자 직접 등록)
- 전화로 접수된 민원 등록 가능
- 민원 종류: 수업 관련, 결제 관련, 기타

## 10. 이탈예측 대시보드

**경로:** `/admin/churn`

AI가 학생별 이탈 위험도를 분석하여 조기에 대응할 수 있도록 합니다.

### 위험도 레벨
| 레벨 | 의미 | 권장 조치 |
|------|------|-----------|
| SAFE (안전) | 이탈 위험 낮음 | 모니터링 지속 |
| WARNING (주의) | 위험 신호 있음 | 학부모 상담 권장 |
| DANGER (위험) | 즉시 대응 필요 | 긴급 상담 요청 |

### 위험도 계산 요소
- 출석률 (연속 결석 횟수)
- 과제 제출률
- 최근 상담 이력
- 결제 연체 여부

### 이탈예측 활용 흐름
1. 이탈예측 대시보드에서 위험 학생 확인
2. 해당 학생 클릭 → 상세 원인 확인
3. 학부모 연락처 확인 후 상담 진행
4. 상담 기록 등록

## 11. 상담 기록

학생 상세 페이지(`/admin/students/[id]`)에서 접근합니다.

### 상담 기록 등록 흐름
1. 학생 상세 페이지 접속
2. "상담 기록" 탭 선택
3. **"상담 추가"** 버튼 클릭
4. 아래 정보 입력:
   - 상담 날짜
   - 상담 방법 (전화 / 문자 / 대면)
   - 상담 내용
5. 저장

└── 월간 통계 검토
