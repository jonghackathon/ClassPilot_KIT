## 완료 항목

### Phase 1 — DB 마이그레이션
- `Academy.code` 필드 추가 (`String @unique`)
  - 기존 행에는 `id` 앞 8자리로 자동 backfill
- `User.email` → `String?` (nullable 전환)
  - 수강생은 이메일 없이 가입/로그인 가능
  - 기존 ADMIN/TEACHER 계정은 영향 없음
- `StudentProfile.studentCode` 필드 추가 (`String?`)
- 마이그레이션 파일: `prisma/migrations/20260413120000_add_student_auth_fields/`
- **DB 적용 완료** (`npx prisma migrate deploy` 성공)

---

### Phase 2 — 로그인 UI 재설계

| 경로 | 내용 |
|------|------|
| `/login` | 역할 선택 화면 (원장·강사 / 수강생 두 카드) |
| `/login/staff` | 이메일+비밀번호 로그인 (기존 기능, 클린 재설계) |
| `/login/student` | 3단계 수강생 로그인 (학원코드 → 반/이름 → PIN) |

- 데모 빠른입력 버튼, 역할별 미리보기 링크 **완전 제거**
- `(auth)/layout.tsx` 좌측 패널: AcadeMind → ClassPilot 브랜드명 통일

---

### Phase 3 — 인증 로직

**NextAuth (`src/lib/auth.ts`)**
- `type: 'staff'` → 이메일+비밀번호 기존 방식
- `type: 'student'` → academyCode + studentId + PIN(bcrypt) 검증
- 학원 소속 + STUDENT 역할 + active 상태 3중 검증

**신규 API 엔드포인트**

| 엔드포인트 | 설명 |
|-----------|------|
| `GET /api/auth/academy?code=DEMO-1234` | 학원코드 조회 → `{ id, name }` |
| `GET /api/auth/students?academyCode=xxx` | 반 목록 반환 |
| `GET /api/auth/students?academyCode=xxx&classId=yyy` | 반별 수강생 목록 (이름 끝 한 글자 마스킹) |

---

### Phase 4 — 미들웨어

- `src/middleware.ts` 신규 생성 (기존 `proxy.ts` 로직 대체 및 확장)
- `/login`, `/login/*` 하위 경로 모두 커버
- 인증 사용자가 `/login/*` 접근 시 역할별 홈으로 redirect
- 미인증 사용자의 보호 경로 접근 시 `/login?callbackUrl=` redirect
- 역할 불일치 접근 시 본인 역할 홈으로 redirect

---

### Phase 5 — 관리자 화면 업데이트

**대시보드 (`/admin/dashboard`)**
- 학원 코드 카드 추가 (수강생 로그인 시 사용 안내 포함)
- `GET /api/academy` 엔드포인트 연동

**학생 상세 (`/admin/students/[id]`)**
- 학번(`studentCode`) 표시 (미발급 안내 포함)
- `email` nullable 처리 (없음 표시)
- **PIN 초기화** 버튼 추가
  - 클릭 시 confirm → `POST /api/users/[id]/reset-pin` → PIN을 `0000`으로 변경
  - 완료/실패 상태 피드백 표시

## 로그인 테스트 방법

### 원장/강사 로그인
1. `/login` → **원장·강사** 클릭
2. 이메일: `admin@academind.kr` / 비밀번호: `1234`

### 수강생 로그인 (시드 실행 후)
1. `/login` → **수강생** 클릭
2. 학원 코드: `DEMO-1234`
3. 반 선택 → 이름 선택 (예: `민○`)
4. PIN: `1234`
