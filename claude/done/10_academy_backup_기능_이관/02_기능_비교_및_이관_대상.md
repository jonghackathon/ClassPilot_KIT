## 2. 신규 이관 기능 (ClassPilot에 완전히 없는 것)

### 🆕 A. 커리큘럼 관리 시스템

**왜 필요:** 과제 생성, 수업 진도, 월간 보고서의 기반 데이터. 이것 없으면 나머지 3개 기능이 구조 없이 동작.

| 항목 | 상세 |
|------|------|
| 계층 구조 | 클래스 → 단계(Stage) → 차시(Lesson) |
| 차시 속성 | 번호(901-1), 제목, 교재, 테마/기법 태그 |
| 프리로드 | 초4~중1 독서논술 커리큘럼 기본 데이터 |
| CRUD | 클래스/단계/차시 추가/수정/삭제 |
| UI | 펼치기/접기 트리, 차시 수 카운트 |
| 연동 대상 | 과제 제목 자동완성, 수업 진도 차시 선택 |
| 적용 페이지 | `/admin/curriculum` (신규) 또는 `/teacher/curriculum` |
| 우선순위 | **높음** — 다른 기능의 기반 |
| 복잡도 | 중~상 |

### 🆕 B. 월간 보고서 + AI 성장 평가

**왜 필요:** 학부모 커뮤니케이션의 핵심. 학원 재원율에 직접 영향. 실제 강사가 매월 사용.

| 항목 | 상세 |
|------|------|
| 보고서 구성 | 학원 로고 + 학생 정보 + 수업 진도 + 출석 현황 + 과제 목록 + AI 성장 평가 |
| AI 성장 평가 | Claude API → 🌱이달의 성장 / ⭐잘하고 있어요 / 📈앞으로의 방향 |
| 입력 데이터 | 학생명/학년/반, 이달 과제+첨삭, 수업 기록, 출석률 |
| 인쇄 | `@media print` 최적화, 페이지 나눔 방지, 프로그레스 바 HTML 렌더링 |
| 저장 | 코멘트 + AI 성장 텍스트 DB 저장 |
| 적용 페이지 | `/teacher/reports` (신규) |
| 우선순위 | **높음** |
| 복잡도 | 높음 (데이터 집계 + Claude API + 인쇄 CSS) |

### 🆕 C. AI 에세이 첨삭 (FeedbackPanel)

**왜 필요:** 강사의 시간 절약 + 학생 피드백 품질 향상. 실제 매일 사용하던 핵심 기능.

| 항목 | 상세 |
|------|------|
| 이미지 첨삭 | 학생 손글씨 사진 → Claude 멀티모달 → 구조화된 피드백 |
| 텍스트 첨삭 | 학생 타이핑 글 → Claude 텍스트 → 동일 구조 피드백 |
| 피드백 구조 | ⭐잘한점 / ✏️교정 / 📝구성피드백 / 📈발전방향 |
| 수동 편집 | AI 결과를 강사가 수정 후 저장 |
| 적용 페이지 | `/teacher/assignments/[id]` 내 패널 또는 모달 |
| 우선순위 | **높음** |
| 복잡도 | 높음 (이미지 업로드 + Claude 멀티모달 + 구조화 출력) |
| 참고 코드 | `academy_backup/src/api/claude.js` — generateFeedback(), generateFeedbackFromText() |

### 🆕 D. 메모/노트 시스템

**왜 필요:** 강사가 수시로 기록하는 학생 특이사항, 학부모 통화 내용, 공지사항 등. 별도 앱(카톡 등)에 흩어지던 정보를 한 곳에.

| 항목 | 상세 |
|------|------|
| 카테고리 | 공지사항(파랑) / 특이사항(주황) / 학생 메모(초록) / 기타(보라) |
| 아카이브 | 1주 경과 시 자동 아카이브, 수동 아카이브/복원 |
| 검색 | 제목 + 내용 통합 |
| UI | 카드형 리스트, 카테고리 필터, 날짜 구분선 |
| 적용 페이지 | `/teacher/memo` (신규) |
| 우선순위 | **중~높음** |
| 복잡도 | 중 (독립적 CRUD, 타이머 로직) |

### 🆕 E. 설정 페이지

**왜 필요:** AI 기능 사용을 위한 API 키 설정, 보고서 인쇄용 학원 로고 관리.

| 항목 | 상세 |
|------|------|
| 기능 | 로고 업로드, Claude API 키 설정, 비밀번호 변경 |
| 저장 | key-value 테이블 (AppSetting) |
| 적용 페이지 | `/admin/settings` (신규) |
| 우선순위 | **중간** (AI 기능 구현 시 필요) |
| 복잡도 | 낮음 |

## 3. 대폭 개선 이관 (ClassPilot에 있지만 기능이 부족한 것)

ClassPilot 현재: 4버튼 (출석/지각/조퇴/결석) UI만 있고 클릭해도 안 됨.
academy_backup에서 가져올 것:

| 기능 | 상세 | 우선순위 |
|------|------|---------|
| **숙제 O/X + 사유** | 출결 옆에 숙제 완료 버튼 + 미완료 시 사유 입력 | 높음 |
| **결석 사유 인라인 입력** | 결석/지각/조퇴 선택 시 사유 텍스트 필드 표시 | 높음 |
| **주별 그리드 뷰** | 월~토 × 학생 그리드, 클릭으로 상태 순환 | 중간 |
| **30일 통계 뷰** | 학생별 출석률, 출석/지각/결석 횟수 요약 | 중간 |
| **[전체 출석] 버튼** | 반 전체를 한 번에 출석 처리 (확인 모달) | 높음 |
| **주 이동 네비게이션** | ← → 화살표로 주 단위 이동 | 높음 |
| **반별 그룹핑** | 시간표 기반 학생 자동 그룹 | 높음 |

### 🔄 B. 과제 관리 — 핵심 워크플로우 부재

ClassPilot 현재: 과제 카드 + 제출 현황 테이블 UI만.

| **과제 유형 3종** | 에세이/독서과제/어휘 — 유형별 색상 배지 | 높음 |

## 5. 실사용 패턴에서 배울 점

### 5.1 강사 일과 흐름 (그대로 적용해야 할 것)

```
앱 열기 → 출결이 기본 페이지 (홈이 아님)
          ↓
수업 전:  출결 입력 + 숙제 확인 (한 화면에서 동시)
          ↓
수업 중:  수업 진도에 내용 기록
          ↓
수업 후:  과제 생성 (자동 생성 또는 수동)
          ↓
첨삭:    사진 → AI 피드백 → 수정 → 저장
          ↓
월말:    보고서 → AI 성장 평가 → 인쇄
```

### 5.2 DB 패턴 (가져올 것)

- **출결 복합 키:** `(date, student_id)` — 자연스러운 중복 방지
- **숙제+출결 통합:** 같은 레코드에 `homework_status`, `homework_note`, `absence_reason`
- **커리큘럼 JSONB:** stages/lessons를 JSON으로 저장 (빠른 읽기, 계층 보존)
- **key-value 설정:** `app_settings(key, value)` — 유연한 설정 저장

### 5.3 UX 패턴 (가져올 것)

- **숙제 O/X가 출결 옆에:** 별도 페이지가 아니라 출결 입력 시 같이
- **결석 사유 인라인:** 결석 선택 즉시 사유 입력란 표시
- **과제 자동 생성:** 수업 진도 기록에서 체크 한 번으로
- **이미지 업로드 → AI 첨삭:** 사진 찍고 바로 피드백
- **인쇄 최적화:** 보고서는 `@media print`로 깔끔하게
