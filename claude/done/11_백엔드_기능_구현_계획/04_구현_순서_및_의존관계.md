# 04. 구현 순서 및 의존관계

> 현재 저장소는 `src/` 루트 구조를 사용한다. 아래 표의 `app/`, `lib/`, `components/` 경로는 실제 구현 시 `src/` 접두어를 붙여 해석하고, 기존 `(dashboard)` 그룹 표기는 현재 라우트 구조인 `src/app/admin|teacher|student` 기준으로 적용한다.

## 의존관계 그래프

```
Step 0: Node fix + Prisma schema + seed
  ↓
Step 1: Auth completion (PrismaAdapter + bcrypt + middleware + logout)
  ↓
Step 2: Core CRUD pattern establishment (Attendance — full cycle: DB→API→SWR→Modal)
  ↓
Step 3: Admin CRUD (Users, Classes, Schedule, Payments, Consultations, Complaints)
  ↓
Step 4: Teacher features (Assignments+Feedback, Curriculum, Progress+WeekNotes, Memo, Bot/FAQ)
  ↓
Step 5: Student features (Assignments submit, Attendance view, Review, QnA)
  ↓
Step 6: AI integrations (Copilot SSE, Essay feedback Claude, Growth report, Recording Whisper, Review generation, Churn batch)
  ↓
Step 7: Polish (Mobile nav, notifications, skeleton UI, error handling, Pretendard font, Vercel deploy)
```

---

## Step 0: 인프라 기초 세팅

### 포함 항목
- Node.js 버전 고정 (`.nvmrc`, `engines` 필드)
- Prisma schema 완성 (`prisma/schema.prisma`) — 모든 모델 정의 + `url = env("DATABASE_URL")`
- `prisma db push` 또는 `prisma migrate dev`로 Supabase DB 반영
- Seed 데이터 작성 (`prisma/seed.ts`)
- 환경변수 정리 (`.env.example` 확장)

### 생성/수정 파일
| 파일 | 작업 |
|------|------|
| `.nvmrc` | 생성 — `20.11.0` |
| `package.json` | 수정 — `engines`, `prisma.seed` 추가 |
| `prisma/schema.prisma` | 수정 — 전체 모델 확정 |
| `prisma/seed.ts` | 생성 — 데모 데이터 |
| `.env.example` | 수정 — `DATABASE_URL`, Auth, Storage, AI 키 템플릿 |

### 선행 조건
- 없음 (최초 단계)

### 결과물
- `DATABASE_URL` 설정 후 `npx prisma db push` 성공, Supabase에 모든 테이블 생성
- `npx prisma db seed` 실행 시 데모 데이터 삽입
- 개발 환경 누구나 동일하게 구성 가능

### 작업 항목 수: **5개**

## 크리티컬 패스 분석

### 최소 데모 가능 경로 (MVP Demo Path)

데모 시연을 위한 **최소한의 경로**는 다음과 같다:

```
Step 0 (1일) → Step 1 (1일) → Step 2 (1일) → Step 3 일부 (2일) → Step 4 일부 (2일)
```

총 **약 7일**로 핵심 데모 가능.

### MVP 데모에 포함되는 것
1. **로그인/로그아웃** — admin, teacher, student 3가지 역할
2. **출석 관리** — 강사가 출석 체크, 목록 조회
3. **반/학생 관리** — admin이 반 생성, 학생 등록
4. **과제 관리** — 강사가 과제 출제, 학생 목록 확인

### MVP 데모에서 제외되는 것 (나중에 추가)
- AI 기능 전체 (Step 6)
- 결제/상담/민원 (Step 3 일부)
- 학생 제출/복습 (Step 5)
- 폴리싱 (Step 7)

**전체 예상: 약 15-16일** (풀타임 1인 기준)

### 병렬화 가능 구간

Step 3의 각 도메인(Users, Classes, Schedule, Payments, Consultations, Complaints)은 서로 독립적이므로 **여러 명이 동시 작업 가능**. Step 4의 강사 기능들도 일부 병렬화 가능 (Assignments vs Curriculum vs Memo).

단, Step 2는 반드시 1명이 완성해야 한다 — 여기서 확립된 패턴을 모든 후속 CRUD가 복제하기 때문.
