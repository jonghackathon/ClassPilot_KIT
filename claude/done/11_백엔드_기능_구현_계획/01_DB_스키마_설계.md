# DB 스키마 설계

**현재:** `prisma/schema.prisma`에 모델 0개 (generator + datasource만)
**목표:** 프론트 데이터 구조 기반 20+ 모델 설계

---

## 0. 현 프로젝트 기준 보정 메모

- Prisma client는 `src/lib/db.ts`를 기준으로 사용한다.
- 인증 세션은 NextAuth v5의 `auth()` 패턴을 기준으로 하며, 학생 본인 조회는 `session.user.id`를 사용한다.
- 과제 타입 enum은 현재 프론트 흐름에 맞춰 `WORKBOOK` / `ESSAY` / `IMAGE`로 통일한다.
- 출결 상태 enum은 `PRESENT` / `LATE` / `EARLY_LEAVE` / `ABSENT`로 통일한다.
- `admin/settings` 라우트는 아직 실제 프로젝트에 없으므로 `AppSetting`은 향후 운영 설정 확장용 모델로 본다.

## 2. Prisma 스키마 초안

```prisma
// ──────────────────────────────────────
// 사용자 & 인증
// ──────────────────────────────────────

model User {
  id            String    @id @default(cuid())
  email         String    @unique
  password      String
  name          String
  role          UserRole
  academyId     String
  active        Boolean   @default(true)
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt

  academy       Academy   @relation(fields: [academyId], references: [id])
  studentProfile StudentProfile?
  enrollments   Enrollment[]
  attendances   Attendance[]
  submissions   Submission[]
  memos         Memo[]
  consultationsAsOwner Consultation[] @relation("ConsultationOwner")
  botQuestions  BotQuestion[]
  reviewSummaries ReviewSummary[]
}

enum UserRole {
  ADMIN
  TEACHER
  STUDENT
}

model Academy {
  id        String   @id @default(cuid())
  name      String
  logoUrl   String?
  createdAt DateTime @default(now())

  users     User[]
  classes   Class[]
  curricula CurriculumClass[]
  settings  AppSetting[]
}

model StudentProfile {
  id        String   @id @default(cuid())
  userId    String   @unique
  grade     String?                       // "초등 4학년", "중등 1학년", "고등 2학년"
  school    String?
  memo      String?
  createdAt DateTime @default(now())

  user      User     @relation(fields: [userId], references: [id])
  parents   ParentContact[]
}

model ParentContact {
  id               String   @id @default(cuid())
  studentProfileId String
  name             String
  phone            String                 // 암호화 저장
  relation         String?                // "부", "모", "기타"
  createdAt        DateTime @default(now())

  studentProfile   StudentProfile @relation(fields: [studentProfileId], references: [id])
}

// ──────────────────────────────────────
// 반 & 시간표
// ──────────────────────────────────────

model Class {
  id        String   @id @default(cuid())
  name      String                        // "수학 A반"
  subject   String?                       // "수학", "영어", "국어", "논술"
  level     String?                       // "기초", "중급", "심화"
  capacity  Int      @default(10)
  academyId String
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  academy     Academy        @relation(fields: [academyId], references: [id])
  teachers    ClassTeacher[]
  enrollments Enrollment[]
  schedules   Schedule[]
  lessons     Lesson[]
  assignments Assignment[]
}

model ClassTeacher {
  id        String   @id @default(cuid())
  classId   String
  teacherId String
  createdAt DateTime @default(now())

  class     Class @relation(fields: [classId], references: [id])

  @@unique([classId, teacherId])
}

model Enrollment {
  id        String   @id @default(cuid())
  classId   String
  studentId String
  active    Boolean  @default(true)
  createdAt DateTime @default(now())
  deletedAt DateTime?                     // soft delete

  class     Class @relation(fields: [classId], references: [id])
  student   User  @relation(fields: [studentId], references: [id])

  @@unique([classId, studentId])
}

model Schedule {
  id        String   @id @default(cuid())
  classId   String
  dayOfWeek Int                           // 1(월)~6(토)
  startTime String                        // "16:00"
  endTime   String                        // "18:00"
  color     String?                       // "indigo", "sky", "violet" 등
  note      String?
  createdAt DateTime @default(now())

  class     Class    @relation(fields: [classId], references: [id])
  lessons   Lesson[]
  weekNotes WeekNote[]
}

model Lesson {
  id         String       @id @default(cuid())
  classId    String
  scheduleId String?
  date       DateTime
  topic      String?
  status     LessonStatus @default(SCHEDULED)
  createdAt  DateTime     @default(now())

  class      Class     @relation(fields: [classId], references: [id])
  schedule   Schedule? @relation(fields: [scheduleId], references: [id])
  attendances Attendance[]
  copilotSessions CopilotSession[]
  recordings RecordingSummary[]
}

enum LessonStatus {
  SCHEDULED
  COMPLETED
  CANCELLED
}

// ──────────────────────────────────────
// 출결
// ──────────────────────────────────────

model Attendance {
  id             String           @id @default(cuid())
  date           DateTime
  studentId      String
  lessonId       String?
  status         AttendanceStatus
  homeworkStatus String?                   // "O" | "X"
  homeworkNote   String?                   // 미완료 사유
  absenceReason  String?                   // 결석/지각/조퇴 사유
  createdAt      DateTime         @default(now())
  updatedAt      DateTime         @updatedAt

  student  User    @relation(fields: [studentId], references: [id])
  lesson   Lesson? @relation(fields: [lessonId], references: [id])

  @@unique([date, studentId])
}

enum AttendanceStatus {
  PRESENT
  LATE
  EARLY_LEAVE
  ABSENT
}

// ──────────────────────────────────────
// 과제 & 제출
// ──────────────────────────────────────

model Assignment {
  id           String         @id @default(cuid())
  classId      String
  title        String
  content      String?
  type         AssignmentType @default(ESSAY)
  dueDate      DateTime?
  teacherNote  String?
  imageUrls    String[]                    // Supabase Storage URLs
  feedback     String?                     // AI 첨삭 결과
  createdAt    DateTime       @default(now())
  updatedAt    DateTime       @updatedAt

  class        Class        @relation(fields: [classId], references: [id])
  submissions  Submission[]
}

enum AssignmentType {
  WORKBOOK
  ESSAY
  IMAGE
}

model Submission {
  id           String   @id @default(cuid())
  assignmentId String
  studentId    String
  content      String?
  aiUsed       Boolean  @default(false)
  aiUsageDetail String?
  submittedAt  DateTime?
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt

  assignment   Assignment          @relation(fields: [assignmentId], references: [id])
  student      User                @relation(fields: [studentId], references: [id])
  history      SubmissionHistory[]

  @@unique([assignmentId, studentId])
}

model SubmissionHistory {
  id           String   @id @default(cuid())
  submissionId String
  content      String
  charCount    Int
  createdAt    DateTime @default(now())

  submission   Submission @relation(fields: [submissionId], references: [id])
}

// ──────────────────────────────────────
// 커리큘럼 & 진도
// ──────────────────────────────────────

model CurriculumClass {
  id        String   @id @default(cuid())
  name      String                        // "초4 - 친절한 설명 Logos"
  stages    Json                          // [{id, name, lessons: [{number, title, book, theme}]}]
  sortOrder Int      @default(0)
  academyId String
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  academy   Academy @relation(fields: [academyId], references: [id])
}

model WeekNote {
  id         String   @id @default(cuid())
  scheduleId String
  date       DateTime
  content    String
  autoAssign Boolean  @default(false)
  createdAt  DateTime @default(now())

  schedule   Schedule @relation(fields: [scheduleId], references: [id])

  @@unique([scheduleId, date])
}

// ──────────────────────────────────────
// 결제 & 상담
// ──────────────────────────────────────

model Payment {
  id        String        @id @default(cuid())
  studentId String
  classId   String
  amount    Int
  status    PaymentStatus @default(UNPAID)
  month     String                        // "2026-04"
  paidAt    DateTime?
  note      String?
  createdAt DateTime      @default(now())

  @@unique([studentId, classId, month])
}

enum PaymentStatus {
  PAID
  UNPAID
  PARTIAL
}

model Consultation {
  id        String           @id @default(cuid())
  studentId String
  ownerId   String                         // 상담 진행자 (강사/운영자)
  type      ConsultationType
  content   String
  createdAt DateTime         @default(now())

  owner     User @relation("ConsultationOwner", fields: [ownerId], references: [id])
}

enum ConsultationType {
  PHONE
  TEXT
  IN_PERSON
}

// ──────────────────────────────────────
// 메모
// ──────────────────────────────────────

model Memo {
  id        String       @id @default(cuid())
  title     String?
  content   String
  category  MemoCategory
  archived  Boolean      @default(false)
  teacherId String
  createdAt DateTime     @default(now())
  updatedAt DateTime     @updatedAt

enum MemoCategory {

// ──────────────────────────────────────

model Complaint {

  level          ChurnLevel
  attendanceFactor  Int
  homeworkFactor    Int
  accessFactor      Int
  questionFactor    Int
  calculatedAt   DateTime   @default(now())

enum ChurnLevel {
  SAFE
  WARNING
  DANGER

// ──────────────────────────────────────
// AI — 코파일럿

model CopilotSession {
  id        String   @id @default(cuid())

  status    String   @default("ACTIVE")    // ACTIVE | COMPLETED
  createdAt DateTime @default(now())

model CopilotQuestion {
  id        String   @id @default(cuid())

  example   String?

// ──────────────────────────────────────
// AI — 녹음 요약
// ──────────────────────────────────────

model RecordingSummary {
  id           String   @id @default(cuid())
  lessonId     String
  audioUrl     String?
  transcript   String?
  summary      String?
  questions    String?
  nextPoints   String?
  status       String   @default("PROCESSING") // PROCESSING | COMPLETED | FAILED
  progress     Int      @default(0)
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt

  lesson       Lesson @relation(fields: [lessonId], references: [id])
}

// ──────────────────────────────────────
// AI — Q&A 봇
// ──────────────────────────────────────

model BotQuestion {
  id           String   @id @default(cuid())
  studentId    String
  classId      String?
  question     String
  aiAnswer     String?
  teacherAnswer String?
  helpful      Boolean?
  status       String   @default("PENDING") // PENDING | AI_ANSWERED | TEACHER_ANSWERED
  createdAt    DateTime @default(now())

  student      User @relation(fields: [studentId], references: [id])
}

model BotFAQ {
  id       String   @id @default(cuid())
  classId  String?
  question String
  answer   String
  createdAt DateTime @default(now())
}

// ──────────────────────────────────────
// AI — 복습 요약
// ──────────────────────────────────────

model ReviewSummary {
  id        String   @id @default(cuid())

  preview   String?                        // 다음 수업 예습
  readAt    DateTime?
  createdAt DateTime @default(now())

  student   User @relation(fields: [studentId], references: [id])
}

// ──────────────────────────────────────
// 시스템 설정

model AppSetting {
  key       String  @id

  academyId String

  academy   Academy @relation(fields: [academyId], references: [id])
}
```

## 2-1. 실제 Prisma 작성 시 추가 보정 필수 항목

| 항목 | 보정 내용 |
|------|----------|
| `StudentProfile` | API 명세와 맞추려면 `birthDate` 필드 추가 검토 |
| `ClassTeacher` | `teacherId`만 두지 말고 `teacher User @relation(...)` 및 `User` 역참조 필드 추가 |
| `Payment` | `student`, `class` relation 명시 필요 |
| `Consultation` | `student User @relation(...)` 추가 필요 |
| `ReportData` | `student User @relation(...)` 추가 필요 |
| `Complaint` | `student User @relation(...)` 추가 필요 |
| `ChurnPrediction` | `student User @relation(...)` 추가 필요 |
| `BotQuestion` | `class Class? @relation(...)` 관계 보강 필요 |
| `BotFAQ` | `class Class? @relation(...)` 관계 보강 필요 |
| `ReviewSummary` | `lesson Lesson? @relation(...)` 관계 보강 필요 |

위 항목들은 현재 문서 초안에서 의도는 드러나지만 Prisma 코드로 바로 옮기기엔 relation 정의가 덜 닫혀 있는 부분들이다. 실제 구현 전에는 이 표 기준으로 관계를 먼저 보강한 뒤 `prisma validate` 기준으로 다시 정리한다.

## 3. 프론트 하드코딩 → DB 모델 매핑

| 프론트 변수 (파일) | DB 모델 |
|------------------|---------|
| `studentRows` (admin-pages) | User + StudentProfile + Enrollment |
| `teacherRows` (admin-pages) | User (role=TEACHER) + ClassTeacher |
| `classCards` (admin-pages) | Class + Schedule + Enrollment (count) |
| `paymentRows` (admin-pages) | Payment |
| `riskStudents` (admin dashboard) | ChurnPrediction + User |
| `todayClasses` (admin dashboard) | Lesson + Class + Schedule |
| `assignments` (admin dashboard) | Assignment + Submission (count) |
| `consultationRows` (admin-pages) | Consultation |
| `scheduleRows` (admin-pages) | Schedule + Class |
| `complaints` (admin-pages) | Complaint |
| `stats` (teacher dashboard) | Lesson/Attendance/Assignment/BotQuestion count |
| `lessons` (teacher dashboard) | Lesson + Class |
| `questions` (teacher dashboard) | BotQuestion (status=PENDING) |
| `churn` (teacher dashboard) | ChurnPrediction |
| `students` (teacher attendance) | User + Attendance |
| `initialAttendance` (teacher attendance) | Attendance |
| `assignments` (teacher assignments) | Assignment + Submission (count) |
| `rows` (teacher assignment detail) | Submission + SubmissionHistory |
| `cards` (teacher copilot) | CopilotQuestion |
| `recordings` (teacher recording) | RecordingSummary |
| `faqItems` (teacher bot) | BotFAQ |
| `chapters` (teacher progress) | CurriculumClass (stages) |
| `courses` (curriculum-page) | CurriculumClass |
| `memoRows` (memo-page) | Memo |
| `reportRows` (reports-page) | ReportData + User + Attendance + Assignment |
| `monthSummaries` (student attendance) | Attendance (grouped by month) |
| `pending/submitted` (student assignments) | Assignment + Submission |
| `items` (student review) | ReviewSummary |
| `history` (student qna) | BotQuestion |
