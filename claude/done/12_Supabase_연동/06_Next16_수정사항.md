## Next.js 16.2.2 주요 변경사항 (2026-04-12 기준)

**Why:** Next.js 16은 기존 15와 여러 breaking changes 있음. 모르고 쓰면 무한 로딩 등 이슈 발생.
**How to apply:** 신규 파일 생성 또는 관련 기능 수정 시 반드시 아래 사항 확인.

---

## 1. middleware.ts → proxy.ts 이름 변경

Next.js 16에서 `middleware` 파일 컨벤션이 `proxy`로 이름 변경됨 (deprecated).

- `src/middleware.ts` → `src/proxy.ts`로 rename 완료
- export default 방식 유지 가능 (또는 named export `proxy` 함수)
- 마이그레이션 코드모드: `npx @next/codemod@canary middleware-to-proxy .`

현재 `src/proxy.ts` — NextAuth auth() 래퍼로 role-based 인증 가드:
```typescript
export default auth((request) => { ... })
export const config = { matcher: ['/login', '/admin/:path*', '/teacher/:path*', '/student/:path*'] }
```

## 2. Turbopack 컴파일 행(hang) 이슈

Next.js 16 기본값이 Turbopack인데, 일부 상황에서 "/" 라우트 컴파일 시 멈춤 발생.
webpack으로 우회 실행:

```bash
next dev --webpack
```

또는 package.json scripts 수정:
```json
"dev": "next dev --webpack"
```

## 3. Proxy (구 Middleware) 런타임

Next.js 16에서 Proxy는 기본적으로 Node.js 런타임 사용 (Edge 아님).
`runtime` config 옵션은 proxy 파일에서 사용 불가.
