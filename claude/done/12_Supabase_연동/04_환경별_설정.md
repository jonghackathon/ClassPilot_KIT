# 환경별 설정

---

## 1. 환경 구분

| 환경 | DB | 용도 |
|------|-----|------|
| **로컬 (dev)** | Supabase 또는 로컬 Postgres | 개발 + 테스트 |
| **프로덕션 (prod)** | Supabase 클라우드 | 실서비스 |

## 2. 로컬 개발 환경

### 2.1 .env (로컬)

```env
# Database — Supabase 클라우드 (개발용 프로젝트)
DATABASE_URL="postgresql://postgres.[ref]:[password]@aws-0-ap-northeast-1.pooler.supabase.com:6543/postgres?pgbouncer=true"
DIRECT_URL="postgresql://postgres.[ref]:[password]@aws-0-ap-northeast-1.pooler.supabase.com:5432/postgres"

# NextAuth
AUTH_SECRET="로컬용-시크릿-키"
AUTH_URL="http://localhost:3001"
AUTH_TRUST_HOST="true"

# Supabase
NEXT_PUBLIC_SUPABASE_URL="https://xxxxxxxxxxxx.supabase.co"
NEXT_PUBLIC_SUPABASE_ANON_KEY="eyJ..."
SUPABASE_SERVICE_ROLE_KEY="eyJ..."
NEXT_PUBLIC_SUPABASE_BUCKET="classpilot"

# AI (개발 시 선택)
ANTHROPIC_API_KEY="sk-ant-..."
OPENAI_API_KEY="sk-..."
```

### 2.2 로컬 대안: 로컬 Postgres

Supabase 대신 로컬 PostgreSQL을 쓰고 싶다면:

```bash
# Docker로 로컬 Postgres
docker run --name classpilot-db \
  -e POSTGRES_PASSWORD=localdev \
  -e POSTGRES_DB=classpilot \
  -p 5432:5432 \
  -d postgres:15

# .env
DATABASE_URL="postgresql://postgres:localdev@localhost:5432/classpilot"
DIRECT_URL="postgresql://postgres:localdev@localhost:5432/classpilot"
```

또는 현재 설정된 Prisma 로컬 Postgres 유지:
```bash
# 현재 .env에 있는 prisma+postgres://localhost:51213 사용
npx prisma dev  # Prisma 로컬 서버 시작
```

```
NEXT_PUBLIC_SUPABASE_ANON_KEY = eyJ...
SUPABASE_SERVICE_ROLE_KEY = eyJ...
NEXT_PUBLIC_SUPABASE_BUCKET = classpilot
ANTHROPIC_API_KEY      = sk-ant-...
OPENAI_API_KEY         = sk-...
```

### 3.2 Vercel 빌드 설정

```json
// package.json
{
  "scripts": {
    "postinstall": "prisma generate",
    "build": "next build",
    "dev": "next dev -p 3001"
  }
}
```

> `postinstall`에서 `prisma generate`를 실행해야 Vercel에서 Prisma Client가 생성됨.

### 3.3 프로덕션 마이그레이션

```bash
# 로컬에서 프로덕션 DB에 마이그레이션 적용
DATABASE_URL="프로덕션URL" npx prisma migrate deploy
```

> `migrate dev`가 아닌 `migrate deploy` 사용 (프로덕션에서는 안전한 배포만)

## 4. Connection Pooling 설정

### Supabase Pooling 모드

| 모드 | 포트 | 용도 |
|------|------|------|
| Transaction | 6543 | 서버리스 (Next.js API, Vercel) — **기본 사용** |
| Session | 5432 | 마이그레이션, seed |

### Prisma에서 Pooling 사용

```prisma
datasource db {
  provider  = "postgresql"
  url       = env("DATABASE_URL")      // pooling (6543, ?pgbouncer=true)
  directUrl = env("DIRECT_URL")        // direct (5432)
}
```

- **런타임** (API Routes): `DATABASE_URL` (pooling) 사용
- **마이그레이션** (`prisma migrate`): `DIRECT_URL` (direct) 자동 사용

### 연결 제한 설정

```env
# CONNECTION_LIMIT을 URL에 추가 (서버리스 환경)
DATABASE_URL="postgresql://...?pgbouncer=true&connection_limit=5"
```

> Supabase Free: 최대 60 direct connections. Vercel 서버리스는 요청마다 연결을 맺으므로 pooling 필수.

## 5. 보안 체크리스트

```
□ .env 파일이 .gitignore에 포함되어 있는지 확인
□ .env.example에는 실제 값 대신 placeholder만 있는지 확인
□ SUPABASE_SERVICE_ROLE_KEY는 서버 사이드에서만 사용 (NEXT_PUBLIC_ 접두사 없음)
□ Supabase 대시보드에서 Row Level Security 활성화 (Storage 버킷)
□ 프로덕션 AUTH_SECRET은 충분히 긴 랜덤 문자열
□ 데이터베이스 비밀번호는 최소 16자 이상
```

## 6. 전체 실행 순서 (처음부터)

```bash
# 1. Supabase 프로젝트 생성 (대시보드)

# 2. .env 파일에 연결 정보 입력
cp .env.example .env
# → DATABASE_URL, DIRECT_URL, AUTH_SECRET 등 입력

# 3. Prisma 스키마에 url 추가
# prisma/schema.prisma → datasource db { url, directUrl }

# 4. 스키마 검증
npx prisma validate

# 5. Client 생성
npx prisma generate

# 6. 마이그레이션 실행
npx prisma migrate dev --name init

# 7. Seed 실행
npx prisma db seed

# 8. dev 서버 실행
npm run dev -- -p 3001

# 9. 로그인 테스트
# http://localhost:3001/login
# admin@academind.kr / 1234

# 10. API 테스트
# curl http://localhost:3001/api/users (세션 필요)
```
