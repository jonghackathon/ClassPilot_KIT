# 기능별 상세 계획

**작성일:** 2026.04.09  
**원칙:** ClassPilot 디자인 유지 (glass-panel, surface-card, Indigo/Violet), 기능 로직 신규 구현

---

## 1. 구현 원칙

- 디자인 시스템은 기존 ClassPilot 컨벤션 유지 (`glass-panel`, `surface-card`, Indigo-600)
- DB 스키마는 기존 Prisma 모델에 확장 (신규 모델 추가, 기존 모델 컬럼 추가)
- API는 기존 `/api/` 패턴 따름 (NextAuth 세션, Prisma ORM)
- Codex 에이전트로 구현, 복잡한 AI 연동은 Claude가 직접 처리

---

## 2. DB 확장 계획

### 신규 모델

```prisma
model Curriculum {
  id        String   @id @default(cuid())
  name      String
  stages    Stage[]
  academyId String
}

model Stage {
  id           String     @id @default(cuid())
  title        String
  order        Int
  curriculumId String
  lessons      Lesson[]
}

model Lesson {
  id      String @id @default(cuid())
  number  String  // "901-1"
  title   String
  order   Int
  stageId String
}

model MonthlyReport {
  id          String   @id @default(cuid())
  studentId   String
  month       String   // "2026-04"
  aiComment   String?
  teacherNote String?
  createdAt   DateTime @default(now())
}

model Memo {
  id         String      @id @default(cuid())
  title      String
  content    String
  category   MemoCategory
  archived   Boolean     @default(false)
  teacherId  String
  createdAt  DateTime    @default(now())
}

model AppSetting {
  key   String @id
  value String
}
```

### 기존 모델 컬럼 추가

```
Attendance: + homeworkDone Boolean, + homeworkNote String?, + absenceReason String?
Assignment: + type AssignmentType (ESSAY/READING/VOCAB), + imageUrl String?
LessonLog:  + content String, + curriculumLessonId String?
Student:    + memo String?, + scheduleId String?
```

## 3. API 엔드포인트 계획

| 엔드포인트 | 메서드 | 기능 |
|-----------|--------|------|
| `/api/curriculum` | GET/POST | 커리큘럼 목록/생성 |
| `/api/curriculum/[id]` | GET/PATCH/DELETE | 커리큘럼 상세/수정/삭제 |
| `/api/reports/[studentId]` | GET/POST | 월간 보고서 조회/생성 |
| `/api/reports/[id]/ai` | POST | AI 성장 평가 생성 (SSE) |
| `/api/assignments/[id]/feedback` | POST | AI 에세이 첨삭 |
| `/api/memos` | GET/POST | 메모 목록/생성 |
| `/api/memos/[id]` | PATCH/DELETE | 메모 수정/삭제/아카이브 |
| `/api/settings` | GET/PATCH | 설정 조회/수정 |

## 4. 기능별 구현 담당

| 기능 | 담당 | 이유 |
|------|------|------|
| 커리큘럼 CRUD | Codex | 패턴 명확한 CRUD |
| 월간 보고서 UI | Codex | 컴포넌트 조합 |
| 보고서 AI 성장 평가 | Claude | SSE + Claude API 연동 |
| AI 에세이 첨삭 | Claude | 멀티모달 + 구조화 출력 |
| 메모 시스템 | Codex | 독립적 CRUD + 타이머 |
| 설정 페이지 | Codex | 단순 key-value |
| 출결 고도화 | Codex | 기존 패턴 확장 |
| 과제 고도화 | Codex + Claude | 이미지 업로드는 Claude |
| 수업 진도 고도화 | Codex | 기존 패턴 확장 |
