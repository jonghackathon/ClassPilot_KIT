# 추가 요구사항 배경 분석

**작성일:** 2026.04.09  
**목적:** 실사용 환경에서 수집된 피드백을 분석하여 추가 기능 요구사항 도출

---

## 1. 배경

ClassPilot 초기 기획(Step 1~3) 완료 후, 실제 학원 강사와의 인터뷰 및 사용성 검토를 통해 현장에서 꼭 필요한 기능들이 누락되어 있음을 확인했다.

초기 기획은 SaaS의 구조적 뼈대(역할 분리, 인증, 대시보드)에 집중했으나, 강사의 **실제 일과 흐름**과 맞지 않는 부분이 있었다.

---

## 2. 강사 인터뷰에서 확인한 실제 일과 흐름

```
앱 열기 → 출결 화면이 기본 (홈 대시보드보다 자주 씀)
          ↓
수업 전:  출결 입력 + 숙제 확인 (한 화면에서 동시에)
          ↓
수업 중:  수업 진도에 오늘 내용 기록
          ↓
수업 후:  과제 생성 (수업 기록에서 자동 생성)
          ↓
첨삭:    학생 글 사진 찍기 → AI 피드백 → 수정 후 저장
          ↓
월말:    학생별 보고서 생성 → AI 성장 평가 → 학부모에게 인쇄 전달
```

---

## 3. 현황 대비 부족한 기능

### 3.1 완전히 없는 기능 (신규 개발 필요)

| 기능 | 필요한 이유 |
|------|-------------|
| **커리큘럼 관리** | 과제·수업 진도·보고서의 기반 데이터. 없으면 다른 기능도 구조 없이 동작 |
| **월간 보고서** | 학부모 커뮤니케이션 핵심. 재원율에 직접 영향 |
| **AI 에세이 첨삭** | 강사가 매일 쓰는 핵심 기능. 수기 첨삭 시간 대폭 절감 |
| **메모/노트** | 학생 특이사항, 학부모 통화 내용을 한 곳에 기록 |
| **설정 페이지** | AI 기능 API 키, 학원 로고 등 운영 필수 설정 |

### 3.2 있지만 실사용 불가 수준인 기능 (고도화 필요)

| 기능 | 현황 | 문제점 |
|------|------|--------|
| **출결 관리** | 버튼 UI만 존재 | 숙제 확인, 결석 사유, 전체 출석 등 실제 기능 부재 |
| **과제 관리** | 카드 목록 UI만 | 유형 구분, 이미지 업로드, 일괄 생성 없음 |
| **수업 진도** | 정적 프로그레스바 | 실제 수업 내용 기록, 과제 자동 생성 없음 |
| **학생 관리** | 기본 CRUD | 과제 수 표시, 시간표 연동, 메모 필드 없음 |
| **시간표** | 테이블 UI | 시각 캘린더, 학생 배정 기능 없음 |

---

## 4. 추가 요구사항 도출 방식

1. **강사 인터뷰** — 실제 수업 진행 시 어떤 기능을 가장 많이 쓰는지 파악
2. **워크플로우 분석** — 수업 준비→진행→마무리 흐름에서 시스템 지원이 필요한 지점 식별
3. **우선순위 결정** — 재원율·시간 절감·학부모 만족도 기준으로 HIGH/MED/LOW 분류
4. **Claude 설계 검토** — 기능별 DB 스키마·API·UI 구조 설계 후 실현 가능성 확인
