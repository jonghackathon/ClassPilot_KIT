**기준 코드 경로:** `src/app`, `src/components/ui`  
**연관 문서:** `docs/05_화면 설계서/00_목차.md`, `docs/02_기능_요구사항_정의서.md`

---

## 1. 문서 목적

이 문서는 현재 코드 기준으로 프론트엔드가 어디까지 구현되었는지 확인하고,  
아직 구현되지 않은 화면과 공통 기능을 정리하기 위한 현황 문서입니다.

구현 여부는 다음 기준으로 판단했습니다.

- **구현됨**: 실제 레이아웃/카드/버튼/내비게이션/텍스트 구조가 존재하고 화면 형태가 잡혀 있음
- **부분 구현**: 화면 골격은 있으나 실제 기능 연결 또는 세부 화면이 비어 있음
- **미구현**: 페이지 제목과 `"구현 예정"` 텍스트만 있는 placeholder 상태

---

## 2. 전체 요약

현재 프론트엔드는 **서비스 셸(shell)과 핵심 첫 진입 화면 일부만 구현된 상태**입니다.

### 수치 요약

| 항목 | 수량 | 상태 |
|------|------|------|
| `page.tsx` 총 페이지 수 | 30개 | 전체 화면 기준 |
| `layout.tsx` 총 레이아웃 수 | 5개 | 루트/인증/운영자/강사/수강생 |
| `"구현 예정"` placeholder 페이지 수 | 25개 | 대부분 상세/목록 화면 |
| 실제 UI가 잡힌 페이지 수 | 5개 | `/`, `/login`, `admin dashboard`, `teacher dashboard`, `student home` |
| 공용 UI 컴포넌트 수 | 20개 | `src/components/ui` 기준 |

### 한 줄 결론

- **브랜딩, 전역 스타일, 역할별 레이아웃, 첫 화면 톤앤매너는 시작됨**
- **실제 업무 화면 대부분은 아직 placeholder 상태**
- **기획/화면설계서 기준으로 보면 Phase 1 MVP의 다수가 아직 미구현**

## 7. 현재 프론트의 구조적 상태 진단

## 7.1 잘 되어 있는 부분

- 전역 스타일 톤이 정리됨
- 역할별 레이아웃 방향성이 분리됨
- 로그인/운영자/강사/수강생 첫 화면의 분위기 차별화가 시작됨
- 모바일용 수강생 탭, 운영자 사이드바, 강사 탭 네비 구조가 존재함

## 7.2 아직 부족한 부분

- 대부분의 실제 업무 화면이 없음
- 데이터 fetching, loading, empty, error 상태가 없음
- CTA 버튼이 실제 업무 플로우까지 이어지지 않음
- 설계서에 있는 표/폼/필터/모달/상세 화면이 코드에 반영되지 않음
- 재사용 가능한 도메인 컴포넌트 계층이 아직 없음

- 대시보드에서 이어지는 실제 상세 기능으로 연결 가능

- `teacher/assignments`

- `student/assignments`
- `student/assignments/[id]`

이유:

- 강사와 수강생 양방향 플로우를 연결해줌
- 과제는 서비스 체감도가 높음

## 9. 최종 결론

현재 프론트엔드는 **브랜드 방향성과 역할별 셸은 잡혔지만, 실제 기능 화면은 아직 대부분 비어 있는 상태**입니다.

정리하면:

- 구현 완료에 가까운 영역: 전역 스타일, 로그인 UI, 역할별 레이아웃 셸
- 부분 구현 영역: 운영자 대시보드, 강사 홈, 수강생 홈
- 미구현 영역: 나머지 대부분의 목록/상세/폼/업무 화면

즉, 지금 단계는 **"프론트 구조와 분위기를 만든 단계"**에 가깝고,  
다음 단계부터는 **실제 기능 화면을 설계서 순서대로 채워 넣는 구현 단계**로 넘어가야 합니다.
