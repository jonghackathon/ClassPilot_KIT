# 화면별 설계 대비 누락 검토 (2차)

**프로젝트명:** AcadeMind
**작성일:** 2026.04.08
**검토 기준:** `docs/05_화면 설계서/` 전체 vs `src/` 실제 코드 (2차 구현 후)
**이전 대비:** 1차 검토 시 placeholder였던 25개 화면 중 대부분이 실제 UI로 구현됨

---

## 1. 전체 현황 요약

### 구현 수치

| 항목 | 1차 | 2차 (현재) | 변화 |
|------|-----|----------|------|
| 실제 UI 페이지 | 5개 | **30개 전체** | +25 |
| placeholder 페이지 | 25개 | **0개** | -25 |
| 공용 도메인 컴포넌트 | 0개 | **4개 파일** (common, admin, teacher, student) | +4 |
| Link 연결된 네비게이션 | 일부 | **전체 연결됨** | 대폭 개선 |
| StatCard 드릴다운 | 없음 | **Link 래핑 완료** | 해결 |

### 아키텍처 변화

```
[이전]  page.tsx에 직접 UI 구현 (또는 placeholder)
[현재]  page.tsx는 5줄 래퍼 → components/frontend/*.tsx에서 실제 UI
        common.tsx에서 SurfaceCard, PageHero, ActionButton 등 공용 컴포넌트 제공
```

### 한 줄 결론

**모든 화면이 UI 수준으로 구현되었고, 페이지 간 네비게이션도 전체 연결됨.**
남은 것은 인터랙티브 기능(모달, 폼 제출, 상태 토글)과 백엔드 연동.
