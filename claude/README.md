# Claude Code 워크플로우

이 폴더는 Claude Code 작업을 체계적으로 관리하기 위한 워크플로우 구조입니다.

## 폴더 구조

| 폴더 | 상태 | 설명 |
|------|------|------|
| `backlog/` | 대기 | 아직 시작하지 않은 작업 |
| `in-progress/` | 진행 중 | 현재 작업 중인 항목 |
| `done/` | 완료 | 완료된 작업 |
| `archive/` | 보관 | 과거 문서 및 더 이상 활성화되지 않는 자료 |
| `docs/` | 참고 | 프로젝트 기획/설계 문서 |
| `templates/` | 템플릿 | 작업 문서 작성용 템플릿 |

## 작업 흐름

```
backlog → in-progress → done
                          ↓
                      archive (필요 시)
```

## 작업 문서 작성 규칙

1. 파일명: `{번호}_{작업명}.md` (예: `01_로그인_구현.md`)
2. 템플릿 사용: `templates/` 폴더의 템플릿을 복사하여 작성
3. 상태 변경 시: 해당 폴더로 파일 이동
4. 완료 시: `done/` 폴더로 이동하고 완료일 기록

## Git 브랜치 연동

- 작업 시작 시: `feature/{작업명}` 브랜치 생성
- 완료 시: `develop` 브랜치로 PR 생성
- 머지 후: `done/`으로 문서 이동
