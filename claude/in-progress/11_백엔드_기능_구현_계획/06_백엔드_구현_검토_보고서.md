## 3. API 패턴 검증

모든 API 라우트가 일관된 패턴을 따르고 있습니다:

```
1. withAuth(roles) → 세션 + 권한 체크
2. getPageParams(request) → 페이지네이션 파라미터
3. 역할별 where 조건 분기 (ADMIN: academyId / TEACHER: classTeacher / STUDENT: studentId)
4. Prisma 쿼리 (include relations)
5. paginatedResponse() 또는 successResponse()
6. 에러 시 errorResponse(code, message, status)
```

**출결 API 검증 (대표 샘플):**
- GET: 역할별 where 분기 (ADMIN→academyId, TEACHER→classTeacher, STUDENT→본인)
- POST: Zod 검증 → upsert (date+studentId unique)
- BULK: 반 전체 학생 일괄 PRESENT
- PATCH: 상태/숙제/사유 개별 수정
