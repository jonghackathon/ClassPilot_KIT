## Step 1: DB 기반 (Prisma 스키마 + Seed)

**왜 먼저:** DB 없으면 API도, 인증도, 데이터 연동도 모두 불가능.

### 작업 내용

| # | 작업 | 산출물 |
|---|------|-------|
| 1 | Prisma 스키마 작성 | `prisma/schema.prisma` — 03_DB_스키마_설계서 기반 |
| 2 | Supabase 프로젝트 생성 | DATABASE_URL 확보 |
| 3 | `prisma migrate dev` 실행 | 테이블 생성 |
| 4 | seed 스크립트 작성 | `prisma/seed.ts` — 데모 데이터 (학생 5명, 강사 3명, 반 3개 등) |
| 5 | `prisma db seed` 실행 | 개발용 데이터 투입 |

### 주요 테이블

```
User, Academy, StudentProfile, ParentContact,
Class, ClassTeacher, Enrollment, Schedule, Lesson,
Attendance, AttendanceEditLog,
Curriculum, CurriculumUnit, LessonProgress,
Assignment, Submission, SubmissionHistory,
Payment, Consultation, MakeupClass,
CopilotSession, CopilotQuestion,
RecordingSummary, BotQuestion, BotFAQ,
ReviewSummary, ChurnPrediction, ChurnEvent, Complaint
```

## Step 3: 핵심 CRUD 1개 관통 — 출결 관리

**왜:** 모달, API, 상태관리, SWR, 낙관적 업데이트를 한 번에 패턴 확립. 이후 모든 CRUD가 이 패턴을 따름.

### 작업 내용

| # | 작업 | 설명 |
|---|------|------|
| 1 | API: `GET /api/attendance` | 날짜+반 기준 출결 조회 |
| 2 | API: `POST /api/attendance` | 개별 출결 입력 |
| 3 | API: `POST /api/attendance/bulk` | 전체출석 일괄 처리 |
| 4 | 프론트: 출결 버튼 상태 관리 | useState + 낙관적 업데이트 |
| 5 | 프론트: SWR 연동 | `useSWR('/api/attendance?date=...&classId=...')` |
| 6 | 프론트: [전체출석] 확인 모달 | shadcn Dialog 사용 |
| 7 | 프론트: 요일/주 필터 동작 | 날짜 선택 → URL 쿼리 → SWR 재요청 |

### 이 단계에서 확립하는 패턴

```typescript
// API 패턴
export async function GET(req: NextRequest) {
  const session = await auth()
  if (!session) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  // 역할별 데이터 범위 필터
  // Prisma 쿼리
  // 표준 응답 형식 { success, data, error }
}

// SWR 패턴
const { data, error, isLoading, mutate } = useSWR(key, fetcher)

// 낙관적 업데이트 패턴
mutate(optimisticData, false) → API 호출 → 성공 시 revalidate / 실패 시 rollback

// 모달 패턴
const [open, setOpen] = useState(false)
<Dialog open={open} onOpenChange={setOpen}>...</Dialog>
```
