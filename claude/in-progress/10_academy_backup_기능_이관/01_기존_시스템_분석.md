# 기존 시스템(academy_backup) 분석

**작성일:** 2026.04.09
**분석 대상:** `/academy_backup/` — 실제 강사가 사용하던 학원 관리 앱
**목적:** ClassPilot에 가져올 기능 식별을 위한 기존 시스템 전수 분석

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 이름 | 지혜의숲 독서논술 학원 관리 시스템 |
| 스택 | React 19 + Tailwind 3 + Supabase (PostgreSQL) |

## 2. 기능 목록 (10개 페이지)

### 2.1 로그인 (LoginPage.js — 68줄)
- 비밀번호 기반 인증 (기본값 "1234")
- 30일 세션 토큰 (쿠키 + Supabase app_settings)
- 자동 로그인, 비밀번호 변경

### 2.2 학생 관리 (StudentsPage.js — 377줄 + StudentModal.js — 268줄)
- CRUD (추가/수정/삭제)
- 학년 필터 (초1~초6, 중1~중3)
- 반(class) 필터 (시간표 연동)
- 이름 검색 + 정렬 (이름/학년/등록일)
- 학생별 과제 수 표시
- 학생 메모 필드
- 학생 카드에서 과제 페이지 바로 이동

### 2.3 출결 관리 (AttendancePage.js — 854줄)
- **3가지 뷰 모드:**
  - 요일별 (daily): 선택한 주의 요일별 출결 입력
  - 주별 (weekly): 월~토 × 학생 그리드, 클릭으로 상태 순환
  - 통계 (summary): 30일 학생별 출석률/횟수
- **4가지 출결 상태:** 출석/지각/조퇴/결석
- **숙제 체크:** O/X 버튼 + 미완료 사유 입력
- **결석 사유:** 결석/지각/조퇴 시 사유 인라인 입력
- **반별 그룹핑:** 시간표 기반 학생 그룹
- **[전체 출석]:** 반 전체를 한 번에 출석 처리
- **주 이동:** ← → 화살표로 주 단위 네비게이션

### 2.4 과제 관리 (AssignmentsPage.js — 641줄 + AssignmentForm.js — 381줄)
- **과제 유형 3종:** 에세이 / 독서과제 / 어휘
- **학생 선택:** 카드 그리드 (반별 그룹핑, 복수 선택)
- **일괄 과제 생성:** 선택한 학생 전원에게 동시 생성
- **커리큘럼 연동:** 과제 제목에 커리큘럼 차시 자동완성
- **이미지 업로드:** Supabase Storage, 드래그&드롭, 카메라, 라이트박스, 회전
- **뷰 모드 2종:** 리스트형 / 폴더형 (유형별 그룹)
- **일괄 작업:** 체크박스 선택 → 일괄 삭제
- **유형별 필터링**

### 2.5 AI 피드백 (FeedbackPanel.js — 585줄)
- **이미지 기반 첨삭:** 학생 손글씨 사진 → Claude 멀티모달 분석
- **텍스트 기반 첨삭:** 학생 타이핑 글 → Claude 텍스트 분석
- **구조화된 출력:**
  - ⭐ 잘한 점 (2~3가지)
  - ✏️ 맞춤법·문법 교정 (원문 → 수정안)
  - 📝 내용·구성 피드백 (구조/논리/표현)
  - 📈 발전 방향 (구체적 조언 2~3가지)
- **피드백 수동 편집:** AI 결과를 강사가 수정 가능
- **피드백 저장:** DB에 영구 저장

### 2.6 시간표 (SchedulePage.js — 570줄)
- 주간 시각 캘린더 (9:00~22:00 × 월~토)
- 수업 블록: 요일/시작시간/종료시간/제목/색상/메모
- **8가지 색상:** indigo/blue/teal/green/purple/pink/orange/rose
- 수업 블록 내 학생 배정 (체크박스)
- 제목 자동완성 (커리큘럼 클래스명)
- 시간 충돌 시각 표시

### 2.7 커리큘럼 관리 (CurriculumPage.js — 323줄)
- **계층 구조:** 클래스 → 단계(Stage) → 차시(Lesson)
- **프리로드 데이터:** 초4 Logos, 초5 Classic1000, 초6 Classic1300, 중1 EGO 등
- **차시 속성:** 번호(901-1), 제목, 교재, 테마/기법 태그
- 클래스/단계/차시 CRUD
- 펼치기/접기 트리 UI
- JSONB 저장 (Supabase)

### 2.8 수업 진도 (ClassProgressPage.js — 749줄)
- **주간 캘린더 뷰:** 반별 카드 + 날짜별 수업 내용 기록
- **수업 내용 입력:** 날짜 선택 → "오늘 다룬 내용" 텍스트 입력
- **과제 자동 생성:** ☑ 체크 시 해당 반 전체 학생에게 "[내용] 에세이 쓰기" 과제 자동 생성
- **커리큘럼 차시 선택:** 모달에서 차시 선택 → 수업 내용에 연동
- **수업 칩:** 캘린더에 수업 기록 칩 표시

### 2.9 월간 보고서 (ReportsPage.js — 937줄)
- **학생별 월간 보고서 생성**
- **보고서 구성:**
  - 학원 로고 + 학생 정보 헤더
  - 이달 수업 진도 (weekNotes 집계)
  - 출석 현황 (출석률 바 + 출석/지각/결석 횟수)
  - 과제 목록 (첨삭 완료 여부 포함)
  - AI 성장 평가 (Claude 생성)
- **AI 성장 평가 프롬프트:**
  - 입력: 학생명/학년/반, 과제 목록, 첨삭 내용, 수업 기록, 출석률
  - 출력: 🌱 이달의 성장 / ⭐ 잘하고 있어요 / 📈 앞으로의 방향
- **인쇄 최적화:** `@media print` CSS, 페이지 나눔 방지
- **보고서 저장:** 코멘트 + AI 성장 텍스트 DB 저장

### 2.10 메모 (MemoPage.js — 458줄)
- **4가지 카테고리:** 공지사항 / 특이사항 / 학생 메모 / 기타
- **카테고리별 색상:** 파랑/주황/초록/보라
- **아카이브 시스템:** 1주 경과 시 자동 아카이브, 수동 아카이브/복원
- **검색:** 제목 + 내용 통합 검색
- **카테고리 필터**
- CRUD + 삭제 확인

## 3. DB 스키마 (Supabase)

```sql
-- 학생
students (id, name, grade, class_name, memo, schedule_id, created_at)

-- 과제
assignments (id, student_id, title, type, assigned_date, teacher_note, image_urls[], feedback, created_at)

-- 출결
attendance_records (date, student_id, status, homework_status, homework_note, absence_reason)
  PRIMARY KEY (date, student_id)

-- 시간표
schedule_items (id, day, start_time, end_time, title, note, color)

-- 커리큘럼
curriculum_classes (id, name, stages JSONB, sort_order)

-- 수업 기록
week_notes (schedule_id, week_start, text)
  PRIMARY KEY (schedule_id, week_start)

-- 메모
memos (id, title, content, category, created_at, updated_at, archived)

-- 보고서
report_data (student_id, month_str, comment, growth)
  PRIMARY KEY (student_id, month_str)

-- 설정
app_settings (key, value)
  -- keys: password, session_token, claude_api_key, logo
```

## 4. AI 통합 상세 (claude.js)

### 4.1 에세이 첨삭 — 이미지 기반

```
입력: 학생 손글씨 사진 (base64 또는 URL) + 학생명/학년/과제유형/과제제목
모델: claude-opus-4-6 (멀티모달)
출력: ⭐잘한점 / ✏️교정 / 📝피드백 / 📈발전방향
```

### 4.2 에세이 첨삭 — 텍스트 기반

```
입력: 학생이 타이핑한 글 + 학생명/학년/과제유형/과제제목
모델: claude-opus-4-6
출력: 동일 구조
```

### 4.3 월간 성장 보고서

```
입력: 학생명/학년/반, 이달 과제 목록, 첨삭 내용(최대 3건), 수업 기록, 출석률
모델: claude-opus-4-6
출력: 🌱이달의 성장 / ⭐잘하고 있어요 / 📈앞으로의 방향
```

## 5. 실사용 패턴 (강사 워크플로우)

강사가 실제로 사용하던 흐름:

```
1. 앱 열기 → 출결 페이지 (기본 페이지)
2. 수업 시작 전: 출결 입력 + 숙제 확인 (O/X)
3. 수업 중: 수업 진도에 오늘 내용 기록
4. 수업 후: 과제 생성 (수업 진도에서 자동 생성 또는 수동)
5. 과제 첨삭: 학생 글 사진 촬영 → 업로드 → AI 피드백 생성 → 수정 → 저장
6. 월말: 보고서 생성 → AI 성장 평가 → 인쇄 → 학부모 전달
7. 수시: 메모에 특이사항 기록
```
