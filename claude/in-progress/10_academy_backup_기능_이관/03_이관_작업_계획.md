### A2. 출결 기능 보강

**대상 페이지:** `/teacher/attendance`
**대상 컴포넌트:** `teacher-pages.tsx` → `TeacherAttendancePage`

**추가할 기능:**

| # | 기능 | 구현 방법 |
|---|------|---------|
| 1 | 숙제 O/X 인터랙션 | 출결 행 옆에 토글 버튼 + 미완료 시 사유 입력 |
| 2 | 결석 사유 인라인 | 결석/지각/조퇴 선택 시 → 텍스트 input 표시 |
| 3 | [전체 출석] 버튼 | 반 헤더에 버튼 + Dialog 확인 모달 |
| 4 | 주 이동 | ← → 버튼으로 주 단위 이동 (useState) |
| 5 | 주별 그리드 뷰 | 월~토 × 학생 테이블, 클릭으로 상태 순환 |
| 6 | 통계 뷰 | 30일 학생별 출석률 요약 카드 |
| 7 | 반별 자동 그룹 | schedule_id 기반 학생 그룹핑 |

**Prisma 모델 보강:**
```prisma
model Attendance {
  id             String   @id @default(cuid())
  date           DateTime
  studentId      String
  lessonId       String?
  status         AttendanceStatus
  homeworkStatus String?           // "O" | "X"
  homeworkNote   String?           // 미완료 사유
  absenceReason  String?           // 결석/지각/조퇴 사유
  createdAt      DateTime @default(now())

  @@unique([date, studentId])     // 복합 유니크 (academy_backup 패턴)
}
```

**참고 코드:** `academy_backup/src/components/AttendancePage.js` (854줄)

### A3. 메모 시스템

**신규 페이지:** `/teacher/memo`
**신규 탭 추가:** 강사 탭 네비에 "메모" 추가

**Prisma 모델:**
```prisma
model Memo {
  id        String   @id @default(cuid())
  title     String?
  content   String
  category  MemoCategory          // NOTICE, NOTABLE, STUDENT_NOTE, OTHER
  archived  Boolean  @default(false)
  teacherId String
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

enum MemoCategory {
  NOTICE        // 공지사항 (파랑)
  NOTABLE       // 특이사항 (주황)
  STUDENT_NOTE  // 학생 메모 (초록)
  OTHER         // 기타 (보라)
}
```

**UI 구성:**
```
PageHero: "메모" + "수업 중 메모, 학생 특이사항, 공지를 한 곳에 정리합니다"

SurfaceCard: 필터 + [+ 메모 작성]
  ├── 카테고리 필터 (StatusBadge: 전체/공지/특이/학생/기타)
  ├── 검색 입력
  └── [아카이브 보기] 토글

메모 카드 리스트:
  ├── 카테고리 배지 + 제목 + 내용 미리보기
  ├── 작성일 / 수정일
  └── [수정] [아카이브] [삭제] 액션
```

**참고 코드:** `academy_backup/src/components/MemoPage.js` (458줄)

## 3. Phase B 상세

### B1. 과제 관리 보강

**대상 페이지:** `/teacher/assignments`, `/teacher/assignments/[id]`

**추가할 기능:**

| # | 기능 | 구현 방법 |
|---|------|---------|
| 1 | 과제 유형 3종 | enum AssignmentType { ESSAY, READING, VOCABULARY } + 색상 배지 |
| 2 | 학생 선택 카드 그리드 | 모달: 반별 그룹핑 + 체크박스 + 과제 수 표시 |
| 3 | 일괄 과제 생성 | 선택 학생 전원에게 Promise.all 생성 |
| 4 | 이미지 업로드 | Supabase Storage + 드래그&드롭 + 미리보기 |
| 5 | 커리큘럼 연동 | 과제 제목 입력 시 차시 자동완성 드롭다운 |

**Prisma 모델 보강:**
```prisma
model Assignment {
  // 기존 필드...
  type       AssignmentType  @default(ESSAY)
  imageUrls  String[]                    // Supabase Storage URL 배열
  feedback   String?                     // AI 첨삭 결과
}

enum AssignmentType {
  ESSAY       // 에세이
  READING     // 독서과제
  VOCABULARY  // 어휘
}
```

**참고 코드:** `academy_backup/src/components/AssignmentsPage.js` (641줄), `AssignmentForm.js` (381줄)

### B2. AI 에세이 첨삭

**대상 페이지:** `/teacher/assignments/[id]` 내 FeedbackPanel

**구현 방법:**

```
과제 상세 페이지:
  ├── 학생별 제출 테이블
  │     └── [첨삭하기] 버튼 클릭 → FeedbackPanel 열기
  │
  └── FeedbackPanel (Sheet 또는 측면 패널):
        ├── 이미지 표시 (업로드된 사진)
        ├── [AI 첨삭 생성] 버튼 → Claude API 호출
        ├── 결과 표시:
        │     ├── ⭐ 잘한 점
        │     ├── ✏️ 맞춤법·문법 교정
        │     ├── 📝 내용·구성 피드백
        │     └── 📈 발전 방향
        ├── [수정] → 텍스트 편집 가능
        └── [저장] → DB에 피드백 저장
```

**API 라우트:** `POST /api/ai/feedback`
- 입력: studentId, assignmentId, images[] 또는 text
- Claude API 호출 (멀티모달 또는 텍스트)
- 응답: 구조화된 피드백 텍스트

**참고 코드:**
- `academy_backup/src/components/FeedbackPanel.js` (585줄)
- `academy_backup/src/api/claude.js` — generateFeedback(), generateFeedbackFromText()

### B3. 수업 진도 보강

**대상 페이지:** `/teacher/progress`

**추가할 기능:**

| # | 기능 | 구현 방법 |
|---|------|---------|
| 1 | 날짜별 수업 내용 기록 | 날짜 선택 → 텍스트 입력 → WeekNote 저장 |
| 2 | 과제 자동 생성 | ☑ "과제 자동 생성" 체크 → 반 전체 학생에게 과제 생성 |
| 3 | 캘린더 뷰 | 날짜 셀에 수업 기록 칩 표시 |
| 4 | 커리큘럼 차시 선택 | 모달에서 차시 선택 → 수업 내용 연동 |

**Prisma 모델:**
```prisma
model WeekNote {
  id         String @id @default(cuid())
  scheduleId String
  date       DateTime
  content    String
  autoAssign Boolean @default(false)  // 과제 자동 생성 여부
  createdAt  DateTime @default(now())

  @@unique([scheduleId, date])
}
```

**참고 코드:** `academy_backup/src/components/ClassProgressPage.js` (749줄)

## 4. Phase C 상세

### C1. 월간 보고서

**신규 페이지:** `/teacher/reports`

**데이터 집계:**
```
1. 해당 월의 출결 데이터 → 출석률 계산
2. 해당 월의 과제 목록 + 첨삭 여부
3. 해당 월의 수업 기록 (WeekNote)
4. Claude API → 성장 평가 생성
5. 보고서 렌더링 + 인쇄
```

**Prisma 모델:**
```prisma
model ReportData {
  id        String @id @default(cuid())
  studentId String
  monthStr  String                    // "2026-04"
  comment   String?                   // 강사 코멘트
  growth    String?                   // AI 성장 평가 텍스트
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@unique([studentId, monthStr])
}
```

**참고 코드:** `academy_backup/src/components/ReportsPage.js` (937줄)
