### 3.3 Prisma 스키마 추가 모델

현재 모델 0개에서 아래 모델 추가:

```prisma
// ── 커리큘럼 ──
model CurriculumClass {
  id        String   @id @default(cuid())
  name      String                          // "초4 - 친절한 설명 Logos"
  stages    Json                            // [{id, name, lessons: [{number, title, book, theme}]}]
  sortOrder Int      @default(0)
  academyId String
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

// ── 메모 ──
model Memo {
  id        String       @id @default(cuid())
  title     String?
  content   String
  category  MemoCategory                    // NOTICE / NOTABLE / STUDENT_NOTE / OTHER
  archived  Boolean      @default(false)
  teacherId String
  createdAt DateTime     @default(now())
  updatedAt DateTime     @updatedAt
}

enum MemoCategory {
  NOTICE                                    // 공지사항
  NOTABLE                                   // 특이사항
  STUDENT_NOTE                              // 학생 메모
  OTHER                                     // 기타
}

// ── 수업 기록 (진도용) ──
model WeekNote {
  id         String   @id @default(cuid())
  scheduleId String
  date       DateTime
  content    String                         // 수업 내용
  autoAssign Boolean  @default(false)       // 과제 자동 생성 여부
  createdAt  DateTime @default(now())

  @@unique([scheduleId, date])
}

// ── 월간 보고서 ──
model ReportData {
  id        String   @id @default(cuid())
  studentId String
  monthStr  String                          // "2026-04"
  comment   String?                         // 강사 코멘트
  growth    String?                         // AI 성장 평가 텍스트
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@unique([studentId, monthStr])
}
```

```
  └── @media print CSS로 깔끔한 A4 출력
```

**데이터 집계 흐름:**
```
1. GET /api/reports?studentId=X&month=2026-04
2. 서버에서 집계:
   - Attendance: 해당 월의 출결 데이터 → 출석률
   - Assignment: 해당 월의 과제 목록 + 피드백 여부
   - WeekNote: 해당 월의 수업 기록
3. POST /api/ai/growth-report → Claude API
   - 입력: 학생 정보 + 과제 + 첨삭 + 수업기록 + 출석률
   - 출력: 성장/강점/방향 마크다운
4. POST /api/reports/[studentId] → 코멘트 + AI 성장 저장
```

```
  이름 | 학년 | [출석][지각][조퇴][결석] | 제출 StatusBadge

변경 후:
  이름 | 학년 | [출석][지각][조퇴][결석] | 숙제 [O][X] | 사유 입력
                                            ↑              ↑
                                         토글 버튼      결석/지각/조퇴 선택 시
                                         (emerald/rose)  인라인 텍스트 input 표시
```

**인터랙션:**
- 출결 버튼: 클릭 → 해당 상태 선택 (토글, 재클릭 시 해제)
- 숙제 O: `bg-emerald-600 text-white`
- 숙제 X: `bg-rose-600 text-white` → 우측에 사유 input 표시
- 결석/지각/조퇴 선택 시: 해당 행 하단에 "사유:" input 표시

## 6. 의존 관계

```
기능 1 (커리큘럼) ─────→ 기능 10 (진도: 차시 선택)
                  └───→ 기능 8 (과제: 제목 자동완성)
                  └───→ 기능 3 (보고서: 수업 진도 집계)

기능 5,6 (출결 보강) ──→ 기능 3 (보고서: 출석률 집계)

기능 7 (이미지 업로드) ─→ 기능 2 (AI 첨삭: 이미지 입력)

기능 2 (AI 첨삭) ──────→ 기능 3 (보고서: 첨삭 내용 참조)

기능 10 (진도) ────────→ 기능 8 (과제 자동 생성)
                  └───→ 기능 3 (보고서: 수업 기록 집계)
```

**따라서 구현 순서:**
```
1단계: 기능 1 (커리큘럼) + 기능 4 (메모) ← 독립적, 병렬 가능
2단계: 기능 5,6 (출결 보강) + 기능 9 (과제 3유형) ← 독립적
3단계: 기능 7 (이미지 업로드) + 기능 11 (시간표 학생 배정)
4단계: 기능 8 (과제 일괄 생성) + 기능 10 (진도 보강) ← 커리큘럼 의존
5단계: 기능 2 (AI 첨삭) ← 이미지 업로드 의존
6단계: 기능 3 (월간 보고서) ← 모든 데이터 의존
```
